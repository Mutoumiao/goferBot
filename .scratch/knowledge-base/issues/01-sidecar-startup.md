## What to build

建立 Tauri Rust 与 Node.js Hono Sidecar 的启动、发现和生命周期管理机制，使前端能够在 sidecar 就绪后开始调用 API。

端到端行为：用户启动应用 → Tauri 窗口显示 → 前端展示 Splash Loading（"正在启动服务..."）→ Rust 启动 sidecar Node 进程 → sidecar 启动 Hono 服务、初始化 SQLite → sidecar 将监听端口写入 `.sidecar-port` → Rust 通过 Tauri event `sidecar-ready` 携带端口通知前端 → 前端 `sidecarClient` 收到端口后解除 Loading，渲染完整 Vue UI。若 30 秒内未就绪，前端显示错误页并提供"重试"按钮。若 sidecar 崩溃，Rust 自动重启（递增端口）并通过 `sidecar-restarted` event 通知前端更新端口。

## Agent Brief

**Category:** enhancement
**Summary:** 建立 Tauri Rust 与 Node.js Hono Sidecar 的启动、发现和生命周期管理机制，使前端在 sidecar 就绪后才能调用 API。

**Current behavior:**
项目尚未实现 sidecar 机制。前端 Vue 应用直接由 Tauri 加载，没有本地 HTTP 服务层，也没有 SQLite 数据库后端。

**Desired behavior:**
应用启动时，Rust 主进程应自动启动一个 Node.js sidecar 进程（运行 Hono HTTP 服务）。sidecar 就绪后通过 Tauri event 通知前端，前端才解除 Loading 状态并渲染主 UI。sidecar 崩溃时 Rust 应自动重启它并通知前端更新端口。

**Key interfaces:**
- `sidecar-ready` / `sidecar-restarted` — Tauri event 名称，payload 携带 `port: number`
- `get_sidecar_port()` — Rust IPC 命令，前端可主动查询当前 sidecar 端口
- `sidecarClient` — 前端 fetch 封装，自动感知端口变化并重试请求
- `.sidecar-port` — sidecar 写入的端口文件，位于应用数据目录
- `GET /health` — sidecar 健康检查路由，返回 HTTP 200

- [x] Rust 成功启动 sidecar 进程，监控其生命周期（PID 跟踪）
- [x] sidecar 默认端口 11451，若被占用则自动递增，端口写入 `.sidecar-port`
