## What to build

补齐知识库管理页的右键菜单操作和文件操作功能，并补充回收站页面入口。用户在测试 #03 时发现应用缺少以下基础交互：知识库列表无右键菜单（无法置顶、修改资料、删除）、文件区域无右键菜单（无法新建文件夹、重命名、移动、复制、删除）、无回收站入口。

端到端行为：用户在知识库列表右键某个知识库 → 弹出自定义菜单 → 可选"置顶"（直接 toggle 状态）、"修改资料"（弹窗修改名称和 MDI 图标）、"删除"（弹窗确认后移入回收站）。在文件区域空白处右键 → 菜单显示"新建文件夹"（创建后直接进入行内重命名编辑状态，默认名 `未命名文件夹_短时间戳`）。在文件/文件夹上右键 → 菜单显示"重命名"（行内编辑，锁定扩展名，失焦保存）、"移动到..."/"复制到..."（弹窗：左栏选知识库，右栏仅显示文件夹，顶部面包屑导航，底部"移动至此"/"复制至此"按钮）、"删除"（永久删除，红色警告二次确认）。左侧知识库列表底部有"回收站"固定入口，点击进入回收站页面，显示已删除知识库列表，支持恢复（同名冲突重命名为"<原名>-副本"）。

## Agent Brief

**Category:** enhancement
**Summary:** 补齐知识库管理页的右键菜单操作、文件操作（新建文件夹、重命名、移动、复制、删除）和回收站入口。

**Current behavior:**
#03 已实现知识库 CRUD、文件导入、资源管理器视图、面包屑导航和回收站后端机制，但前端缺少：
1. 知识库列表的右键菜单（无法置顶、修改资料、删除）
2. 文件区域的右键菜单（无法新建文件夹、重命名、移动、复制、删除）
3. 回收站页面入口（用户删除知识库后无处可恢复）

**Desired behavior:**
知识库列表和文件区域均支持自定义右键菜单，覆盖日常文件管理的全部基础操作。回收站通过左侧列表底部入口访问，支持查看和恢复已删除知识库。

**Key interfaces:**
- `PATCH /knowledge-bases/:id` — sidecar API，修改知识库名称/图标/置顶状态
- `POST /knowledge-bases/:id/folders` — sidecar API，新建文件夹
- `PATCH /knowledge-bases/:id/files/:path` — sidecar API，重命名文件（只改文件名）
- `POST /files/move` / `POST /files/copy` — sidecar API，跨库移动/复制文件
- 自定义 Vue 右键菜单组件 — 知识库列表和文件区域共用或分别实现
- 移动/复制弹窗 — 左栏知识库列表 + 右栏文件夹列表 + 面包屑导航
- 回收站页面 — 已删除知识库列表 + 恢复操作

**Acceptance criteria:**
- [ ] `knowledge_bases` 表增加 `is_pinned`、`sort_order`、`icon` 字段
- [ ] 知识库列表右键菜单：置顶、修改资料（弹窗）、删除（移入回收站）
- [ ] 文件区域空白处右键：新建文件夹（默认名带时间戳，立即编辑）
- [ ] 文件/文件夹右键：重命名（行内编辑）、移动到、复制到、删除（永久删除）
- [ ] 移动/复制弹窗：左栏选知识库，右栏仅文件夹，面包屑导航，底部确认按钮
- [ ] 删除弹窗差异化：知识库"移入回收站"，文件红色"永久删除"
- [ ] 左侧知识库列表底部增加"回收站"入口，点击打开回收站页面
- [ ] 回收站页面支持恢复已删除知识库（同名冲突重命名为"<原名>-副本"）
