## 7. 复杂模块

### 7.4 RBAC 权限系统

**数据来源**：[permission.service.ts](file:///d:/projects/ai-stared-project/knowledge-base/packages/server/src/modules/admin/services/permission.service.ts)、[roles.guard.ts](file:///d:/projects/ai-stared-project/knowledge-base/packages/server/src/auth/guards/roles.guard.ts)、[permission.guard.ts](file:///d:/projects/ai-stared-project/knowledge-base/packages/server/src/auth/guards/permission.guard.ts)、[app.guard.ts](file:///d:/projects/ai-stared-project/knowledge-base/packages/server/src/common/guards/app.guard.ts)、[permission.seeder.ts](file:///d:/projects/ai-stared-project/knowledge-base/packages/server/src/modules/admin/seeders/permission.seeder.ts)

> **v22 重大重构**：删除 `User.role` 字段（String 枚举），新增 `Role` 表正规化 RBAC；超级管理员判定从 `permissions.length >= 20` 改为检查 roles 含 `super_admin`；Cookie 按 app 隔离；新增 AppGuard。

#### 四层 Guard 链

```
JwtAuthGuard (路径感知 Cookie 解析 + 黑名单检测)
  → AppGuard (token.app 匹配端点要求，@AllowApp 装饰器覆盖)
    → RolesGuard (@Roles 装饰器 → roles 数组匹配)
      → PermissionGuard (@Permission 装饰器 → 细粒度权限)
```

- **JwtAuthGuard**: 路径感知 Cookie 解析——Admin 路径读 `goferbot_admin_access_token`，Web 路径读 `goferbot_web_access_token`，通用路径按 `X-App-Context` 请求头优先级读取
- **AppGuard**: 新增守卫，按路径分类判定期望 app（public/admin/web-biz/common），token.app 不匹配返回 403 `APP_MISMATCH`。`@AllowApp('web'|'admin'|'both'|'public')` 装饰器覆盖自动判定
- **RolesGuard**: 读取 `@Roles('admin')` 装饰器 → 检查 `user.roles.includes(roleCode)`
- **PermissionGuard**: 读取 `@Permission('users.read')` 装饰器 → 调用 `PermissionService.hasAnyPermission()`

#### Role 表正规化模型

```
Role (id, code, name, description, app, isSystem, sortOrder, status)
  └──< UserRole (userId, roleCode, app) ──> User
  └──< RolePermission (roleCode, permissionCode)
```

- **预置系统角色**（PermissionSeeder 启动自动创建，`isSystem=true`，不可删除）：
  | roleCode | name | app |
  |----------|------|-----|
  | super_admin | 超级管理员 | admin |
  | admin | 管理员 | admin |
  | user | 普通用户 | web |

- **动态角色发现**：`listRoles()` 通过 `UserRole.groupBy('roleCode')` 动态发现已使用角色 + 预置角色 = 完整列表
- **权限分配**：采用"先 deleteMany 清空 → 再 create 重建"模式更新，更新后清除全部权限缓存

#### PermissionService — 权限缓存与 SUPER_ADMIN 判定

- **Redis 缓存优先**: `getUserPermissions(userId, app)` 先查 `auth:permission:{userId}:{app}` 缓存（TTL=300s），miss 时查 UserRole → Role → RolePermission 联查
- **SUPER_ADMIN 判定**: `roles.includes('super_admin')`（替代旧 `permissions.length >= 20`）
- **缓存失效**: `invalidateUserPermissions(userId, app)` 单用户或 `invalidateAllPermissions()` 全量
- **权限验证**: `hasAnyPermission()` / `hasAllPermissions()` — SUPER_ADMIN 直接返回 true

#### PermissionSeeder — 启动自动初始化

- 实现 `OnModuleInit`，启动时检查 Role 表是否为空
- **幂等 upsert**：使用 Prisma `upsert` 预置角色 + 权限码分配
- **补充逻辑**：运行时检测是否有新增权限码 → 自动追加到 super_admin 角色
- 系统角色标记 `isSystem=true`，不可通过 API 删除
- super_admin 分配通配符 `*` 权限（前端 PermissionMatrix 中显示"全部权限"且不可编辑）

#### App 隔离路径分类

| 路径分类 | 示例 | 期望 token.app |
|---------|------|---------------|
| 公开路径 | `/auth/web/login`, `/auth/admin/login`, `/auth/web/register`, `/auth/public-key`, `/captcha/*` | 无需认证 |
| Admin 专属 | `/admin/*` | `admin` |
| Web 专属 | `/chat/*`, `/knowledge-base/*`, `/session/*`, `/companion/*` | `web` |
| Web Auth | `/auth/web/refresh`, `/auth/web/logout` | `web` |
| Admin Auth | `/auth/admin/refresh`, `/auth/admin/logout` | `admin` |
| 通用路径 | `/auth/me`, `/user/*`, `/settings/*` | 双 app 均可，`X-App-Context` 头决定优先级 |

#### Cookie App 隔离

| App | Access Token Cookie | Refresh Token Cookie |
|-----|--------------------|-----------------------|
| Web | `goferbot_web_access_token` | `goferbot_web_refresh_token` |
| Admin | `goferbot_admin_access_token` | `goferbot_admin_refresh_token` |

前端请求拦截器统一添加 `X-App-Context` 头（web 端设 `web`，admin 端设 `admin`），解决同一浏览器双 Cookie 同时发送时的优先级判定。

#### 角色/权限常量

前端 `PERMISSIONS` 常量（21 个）定义在 `@goferbot/data` 共享包，前后端对齐。分组：dashboard / users / roles / invitations / audit / settings / system。

#### 前后端联动

```
前端 (_authenticated beforeLoad)                        后端 (JwtAuthGuard → AppGuard → PermissionGuard)
  ┌─────────────────────────────┐                        ┌──────────────────────────────────────┐
  │ waitForAuthInit()           │                        │ JWT Cookie 验证（路径感知双 Cookie）   │
  │   → GET /auth/me (单飞)     │    HTTP 200            │ JwtStrategy.validate()               │
  │ hasAnyPermission(...)       │ ◄────────────────────  │ AppGuard (token.app 匹配)             │
  │  └─ route 可访问            │                        │ PermissionGuard → 权限码匹配           │
  │  └─ menu 可见               │                        │  └─ 403 APP_MISMATCH / FORBIDDEN       │
  └─────────────────────────────┘                        └──────────────────────────────────────┘
```
