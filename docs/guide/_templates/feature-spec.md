# 功能规格模板

```markdown
---
issue_id: {prefix}-{NN}
type: feature-spec
status: draft
summary: {功能边界、核心用户故事、关键决策，2-3 句话}
---

# 功能规格：{功能名称}

## 用户故事

作为 {用户类型}，我希望 {操作}，以便 {收益}。

## 边界

- 范围内：{包含的内容}
- 范围外：{明确排除的内容}

## 涉及页面/组件

- {页面/组件 1}
- {页面/组件 2}

## 相关功能

- {上游功能} — 提供 {什么}
- {下游功能} — 消费 {什么}

## 已做决策

| 决策 | 理由 | 可逆？ |
|------|------|--------|
| {决策 1} | {原因} | {是/否} |
```

---

## Frontmatter 字段说明

| 字段 | 说明 | 必填 |
|------|------|------|
| `issue_id` | 对应 issue 编号 | ✅ |
| `type` | 固定值：`feature-spec` | ✅ |
| `status` | draft / review / approved / deprecated | ✅ |
| `summary` | 清晰描述功能边界与核心内容，Agent 据此判断是否需深入阅读 | ✅ |

## 正文填写说明

| 字段 | 说明 |
|------|------|
| `用户故事` | 标准三段式，明确用户、操作、收益 |
| `边界` | 范围外必须明确写出"本期不做"的内容 |
| `涉及页面/组件` | 列出所有需要新建或修改的文件 |
| `相关功能` | 建立功能依赖图，标明数据流向 |
| `已做决策` | 记录不可撤销或高成本的决策 |
