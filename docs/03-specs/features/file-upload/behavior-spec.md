# 行为规格：文件上传组件

> 对应 issue: `f-07-file-upload-component`

---

## 入口

- 文件管理器顶部工具栏的"添加文件"按钮
- 拖拽文件到文件管理器区域

## 初始状态

- 按钮常态："添加文件"图标按钮
- 点击后：打开系统文件选择对话框

## 交互状态

| 状态 | 视觉 | 用户操作 | 系统响应 |
|------|------|----------|----------|
| selecting | 系统文件选择对话框 | 选择文件 | 关闭对话框，显示确认列表 |
| confirming | 文件列表对话框（文件名、大小） | 确认/取消 | 开始上传或取消 |
| uploading | 进度条（百分比）+ 取消按钮 | 点击取消 | 中断上传，移除该项 |
| success | 文件项绿色勾选，自动关闭 | 无 | 文件管理器自动刷新 |
| error | 红色错误提示 + 重试按钮 | 点击重试 | 重新上传 |
| dragover | 文件管理器区域高亮（蓝色边框） | 释放文件 | 触发上传流程 |

## 正常流程

| 步骤 | 用户操作 | 系统响应 | 视觉状态 |
|------|----------|----------|----------|
| 1 | 点击"添加文件" | 打开系统文件选择器 | selecting |
| 2 | 选择 1-N 个文件 | 显示上传确认对话框 | confirming |
| 3 | 点击"开始上传" | 逐个上传文件 | uploading |
| 4 | 上传完成 | 对话框关闭，文件管理器刷新 | success |

## 错误场景

| 场景 | 触发 | 视觉 | 恢复 |
|------|------|------|------|
| 文件过大 | 单个 > 50MB | 列表项红色，提示"超过 50MB" | 移除该项后继续 |
| 类型不支持 | 非 md/txt/pdf | 列表项红色，提示"不支持的格式" | 移除该项后继续 |
| 文件名非法 | 包含 `../` 或不可打印字符 | 列表项红色，提示"文件名非法" | 移除该项后继续 |
| 上传失败 | 网络错误/500 | 错误提示 + 重试按钮 | 点击重试 |
| 拖拽非文件 | 拖拽文本/图片 | 无反应 | 无 |

## 动画与过渡

| 过渡 | 持续时间 | 缓动 | 备注 |
|------|----------|------|------|
| 对话框打开 | 200ms | ease-out | 缩放 + 淡入 |
| 进度条 | 实时 | linear | 宽度百分比变化 |
| 成功勾选 | 300ms | ease | 缩放弹出 |
| 拖拽高亮 | 150ms | ease | 边框颜色变化 |

## 无障碍

- 按钮 aria-label："上传文件"
- 进度条 role="progressbar"，aria-valuenow
- 错误提示 role="alert"

## 响应式行为

| 断点 | 布局变化 |
|------|----------|
| 375px | 对话框全屏 |
| 768px+ | 对话框 480px 宽 |
