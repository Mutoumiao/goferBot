# 功能规格：知识库选择器

## 用户故事
作为使用 GoferBot 进行问答的用户，我希望在对话时通过 `@` 提及快速选择知识库，以便触发 RAG 检索增强回答。

## 边界
- 范围内：
  - 输入框中输入 `@` 触发知识库选择下拉
  - 下拉显示用户所有知识库列表
  - 支持键盘导航（上下箭头、Enter、Esc）
  - 支持多选（复选框形式）
  - 已选知识库以标签形式显示在输入框上方
  - 发送消息时携带 `knowledgeBaseIds`
- 范围外：
  - 知识库权重调整
  - 知识库搜索过滤（列表内）
  - 知识库实时索引状态显示
  - 知识库自动补全（非 `@` 触发方式）

## 涉及页面/组件
- `ChatInput.vue` — 输入框组件，集成 `@` 触发逻辑
- `KbSelector.vue` — 知识库选择下拉组件（替代/扩展现有 `KbMentionDropdown.vue`）
- `KbMentionPill.vue` — 已选知识库标签组件（已存在）
- `useKnowledgeBaseStore` — 知识库列表数据

## 相关功能
- `f-09-chat-page` — 提供输入框和消息发送
- `b-02-knowledge-base-crud-api` — 提供知识库列表 API
- `b-04-chat-sse-api` — 接收 `knowledgeBaseIds` 参数进行 RAG 检索

## 已做决策
| 决策 | 理由 | 可逆？ |
|------|------|--------|
| `@` 触发下拉选择 | 符合用户习惯，与 Slack/Discord/飞书等一致 | 是 |
| 多选后标签显示在输入框上方 | 不占用输入框空间，视觉清晰 | 是 |
| 复选框形式多选 | 明确显示已选状态，避免误操作 | 是 |
| 空知识库时提示"请先创建知识库" | 引导用户操作，避免空状态困惑 | 是 |
