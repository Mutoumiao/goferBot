# 功能规格：聊天页面标签栏

## 用户故事
作为使用 GoferBot 进行多主题问答的用户，我希望在聊天页面内同时打开多个会话标签，以便在不同话题间快速切换而不丢失上下文。

## 边界
- 范围内：
  - Chat 页面内部的会话标签栏（38px 固定高度）
  - 标签的创建、切换、关闭、重命名
  - "首页"标签作为默认入口始终保留
  - 标签横向滚动
  - Pinia Store 管理标签状态
- 范围外：
  - 标签拖拽排序
  - 标签分组
  - 标签右键菜单
  - 全局页面标签（由 vue-router + Sidebar 处理）
  - 标签状态持久化（刷新丢失，后续扩展）
  - 非 Chat 页面的标签（知识库、设置等由 Sidebar 导航）

## 涉及页面/组件
- `ChatView.vue` — 主页面，嵌入标签栏
- `ChatTabBar.vue` — 标签栏组件
- `useChatTabsStore` (`chatTabs.ts`) — 标签状态管理

## 相关功能
- `f-09-chat-page` — 提供 Chat 页面容器和消息流
- `b-03-session-api` — 提供会话创建/查询 API
- `f-12-chat-history` — 从历史记录恢复会话时打开对应标签

## 已做决策
| 决策 | 理由 | 可逆？ |
|------|------|--------|
| 标签状态与会话数据分离 | 避免消息数据重复存储，标签只保存 sessionId 引用 | 是 |
| "首页"标签不可关闭 | 保证始终有一个默认入口创建新会话 | 是 |
| 刷新后标签状态丢失 | MVP 简化，后续可扩展 localStorage 持久化 | 是 |
| 关闭标签切换到左侧相邻 | 符合浏览器标签栏惯例 | 是 |
