# 行为规格：V1 数据导出工具

> 注意：本功能为 CLI 工具，无前端交互。此规格描述命令行交互状态。

## 入口
- 命令：`pnpm export:v1 -- --db <path> --user-id <uuid>`
- 触发：用户在终端手动执行

## 交互状态

| 状态 | 视觉 | 用户操作 | 系统响应 |
|------|------|----------|----------|
| 参数校验 | 无输出 | 输入命令 | 缺少必填参数时打印用法并退出码 1 |
| 数据库检查 | `正在打开数据库...` | 等待 | 文件不存在时打印错误并退出码 2 |
| 导出进行中 | 逐表打印进度（`导出会话... N 条`） | 等待 | 读取 V1 表，转换为 V2 兼容格式，写入 NDJSON |
| 成功 | 统计摘要（表格） | 查看输出文件 | 打印输出路径 |
| 错误 | 错误信息 + 堆栈（仅开发环境） | 修复后重试 | 退出码非 0 |

## 正常流程

| 步骤 | 用户操作 | 系统响应 |
|------|----------|----------|
| 1 | `pnpm export:v1 -- --db ./v1-data.sqlite --user-id <uuid>` | 解析参数 |
| 2 | — | 检查 SQLite 文件存在且可读 |
| 3 | — | 读取 `sessions` 表，转换为 V2 格式 |
| 4 | — | 读取 `messages` 表，关联 session，转换为 V2 格式 |
| 5 | — | 读取 `knowledge_bases` 表，转换为 V2 格式 |
| 6 | — | 写入 `./v1-export/YYYY-MM-DD/` 目录 |
| 7 | — | 打印统计摘要 |

## 错误场景

| 场景 | 触发 | 输出 | 恢复 |
|------|------|------|------|
| 缺参数 | 未传 `--db` 或 `--user-id` | 用法提示 + 退出码 1 | 补充参数 |
| 文件不存在 | SQLite 路径无效 | `数据库文件不存在: <path>` + 退出码 2 | 检查路径 |
| 表不存在 | V1 数据库缺少预期表 | `表 <name> 不存在，跳过` + 继续 | 非致命，部分导出 |
| 写权限不足 | 输出目录不可写 | `无法写入输出目录: <path>` + 退出码 4 | 检查权限 |
| userId 格式错误 | 非 UUID 格式 | `--user-id 必须是合法 UUID` + 退出码 1 | 修正参数 |

## 输出格式

### sessions.ndjson
```json
{"id":"<原id>","userId":"<目标用户>","title":"<标题>","createdAt":"<ISO>","updatedAt":"<ISO>"}
```

### messages.ndjson
```json
{"id":"<原id>","sessionId":"<原sessionId>","role":"<user|assistant>","content":"<内容>","createdAt":"<ISO>"}
```

### knowledge_bases.ndjson
```json
{"id":"<原id>","userId":"<目标用户>","name":"<名称>","createdAt":"<ISO>"}
```
