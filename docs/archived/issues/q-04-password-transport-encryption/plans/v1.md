# q-04 密码传输加密计划 v1

## 现有安全措施（无需改动）

| 措施 | 实现方式 |
|------|---------|
| bcrypt 密码哈希 | `UserService.create()` hash cost 12 |
| Zod 输入校验 | `RegisterDto` / `LoginDto` 密码规则 |
| 速率限制 | `@Throttle` 5/min on register/login |
| API Key 加密 | AES-256-GCM（settings.service.ts） |
| Helmet / CORS / ExceptionFilter | bootstrap.ts + app.module.ts |

## 任务列表

### 任务 1: 后端 — RSA 密码加密服务 + 公钥端点

**新建**：`packages/server/src/auth/crypto/password-encryption.service.ts`
**修改**：`packages/server/src/auth/auth.module.ts`（注册 service）
**修改**：`packages/server/src/auth/auth.controller.ts`（新增 GET /public-key）

- [ ] `PasswordEncryptionService` — 启动时 `generateKeyPairSync('rsa', 4096)`，私钥内存持有
- [ ] `getPublicKeyPem()` — 导出 SPKI PEM（`export({ type: 'spki', format: 'pem' })`）
- [ ] `decrypt(encryptedBase64: string): string` — base64 解码 → `privateDecrypt` OAEP-SHA256 → utf8
- [ ] `GET /api/auth/public-key` — 返回 `{ publicKey, algorithm: 'RSA-OAEP', hash: 'SHA-256' }`
- [ ] 该端点不设限速（auth controller 中设为不限速或单独限制）

### 任务 2: 后端 — DTO 扩展 + AuthService 解密逻辑

**修改**：`packages/server/src/auth/dto/register.dto.ts`
**修改**：`packages/server/src/auth/dto/login.dto.ts`
**修改**：`packages/server/src/auth/auth.service.ts`

- [ ] `registerSchema` — 移除 `password` 必填，改为 `encryptedPassword: z.string().min(1)`
- [ ] `loginSchema` — 同上
- [ ] `AuthService.register()` — 调用 `passwordEncryption.decrypt(dto.encryptedPassword)` → bcrypt
- [ ] `AuthService.login()` — 同上
- [ ] 解密失败抛 `BadRequestException('DECRYPT_FAILED')`

### 任务 3: 前端 — Web Crypto 加密 + Store 集成

**新建**：`packages/webui/src/utils/password-encryption.ts`
**修改**：`packages/webui/src/stores/auth.ts`

- [ ] `fetchPublicKey(): Promise<CryptoKey>` — 请求 `/api/auth/public-key`，缓存模块级变量
- [ ] `encryptPassword(publicKey: CryptoKey, password: string): Promise<string>` — `crypto.subtle.encrypt` RSA-OAEP → base64
- [ ] `clearPublicKeyCache()` — 解密失败时清缓存
- [ ] `authStore.login()` — 调用 `fetchPublicKey()` + `encryptPassword()`，失败时捕获 `DECRYPT_FAILED` 重试 1 次
- [ ] `authStore.register()` — 同上
