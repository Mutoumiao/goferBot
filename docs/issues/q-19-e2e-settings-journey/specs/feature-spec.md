# q-19 功能规格

## 概述

设置持久化和跨模块用户旅程的端到端测试，验证复杂用户场景和跨功能集成。

## 功能需求

### FR-01: 设置持久化测试
- 设置页面加载所有 Tab
- 修改 LLM 提供商配置（API Key、Model、BaseUrl）
- 切换默认提供商
- 修改 Embedding 配置
- 调整温度参数
- 保存后刷新页面验证数据恢复

### FR-02: 新用户入职旅程
- 注册新账号
- 自动登录进入首页
- 创建第一个知识库
- 上传第一个文档
- 新建会话
- 发送第一条消息
- 看到 AI 响应

### FR-03: RAG 检索增强工作流
- 创建专用测试知识库
- 上传包含特定事实的文档
- 等待文档解析索引完成（轮询或超时）
- 在聊天中 @ 引用该知识库
- 发送与文档内容相关的问题
- 验证 AI 响应引用了文档中的信息
