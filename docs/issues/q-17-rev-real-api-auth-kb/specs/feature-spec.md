# 功能规格：q-17 真实 API 版本

## 概述

实现 q-17 剩余的 5 个 pending AC，使用真实后端 API 验证认证流程和知识库生命周期。

## 功能边界

### 范围内

- AC-06：未登录访问保护路由重定向
- AC-08：重复注册相同邮箱返回错误
- AC-12：上传文档到知识库
- AC-15：用户 B 无法操作用户 A 的知识库
- AC-16：上传 txt/md/pdf 三种类型文档

### 范围外

- q-17 已通过的 11 个 AC
- mock API 测试
- E2E UI 测试

## 验收标准

| ID | 标准 | 优先级 |
|----|------|--------|
| AC-06 | 未登录访问 `/chat` 等保护路由，重定向到 `/login` | P1 |
| AC-08 | 重复注册相同邮箱，后端返回 409，前端显示错误 | P1 |
| AC-12 | 上传文档到知识库，状态流转 uploaded → ready | P1 |
| AC-15 | 用户 B 的 token 无法访问用户 A 的知识库（返回 403） | P1 |
| AC-16 | 上传 txt/md/pdf 三种类型文档，均成功索引 | P1 |
| AC-17 | 全部测试使用真实后端 API，非 mock | P0 |
| AC-18 | pnpm type-check 通过 | P0 |
| AC-19 | q-17 原 issue 可关闭（16/16 AC 通过） | P0 |

## 技术约束

- 使用 `TestAppFactory.create(dbUrl, { realMode: true })`
- 每个测试用例独立创建/清理数据
- 文档上传后需等待 Worker 处理完成（最长 60 秒）

## 已做决策

| 决策 | 理由 | 可逆？ |
|------|------|--------|
| 使用真实 API 而非 mock | 原 spec 要求，且 mock 价值有限 | 否 |
| 复用 q-22 的 TestAppFactory | 基础设施已就绪，避免重复 | 是 |
| 在集成测试层实现（非 E2E） | 验证 API 契约，无需浏览器 | 是 |
