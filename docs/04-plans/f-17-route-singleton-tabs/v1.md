# 路由单例标记 实现计划

> **For agentic workers:** 使用 `subagent-driven-development`。步骤用复选框追踪。

**目标：** Tab 单例/多重行为由路由 meta 驱动，替代 Store 内硬编码。

**架构：** 抽取独立 `TAB_ROUTE_CONFIG` 配置表（避免 Store → Router 循环依赖）→ 路由和 Store 各自 import 配置表 → Store 的 `addTab` 读取配置判断去重 → AppSidebar 通过 Store 操作标签。

**技术栈：** Vue 3 + Vue Router + Pinia + TypeScript

### 任务 1: 创建独立 TAB_ROUTE_CONFIG 配置表

**文件：**
- 创建：`packages/webui/src/router/tab-routes.ts`

> 独立配置表避免 Store → Router → Layout → Store 循环依赖。路由和 Store 各自 import 此文件。

- [ ] **步骤 1: 创建 tab-routes.ts**

```typescript
// packages/webui/src/router/tab-routes.ts
import type { TabType } from '@/types'

export interface TabRouteConfig {
  routeName: string
  tabType: TabType
  singleton: boolean
  defaultTitle: string
}

export const TAB_ROUTE_CONFIG: TabRouteConfig[] = [
  { routeName: 'chat',          tabType: 'chat',         singleton: false, defaultTitle: '新会话' },
  { routeName: 'knowledgeBase', tabType: 'knowledgeBase', singleton: true,  defaultTitle: '知识库' },
  { routeName: 'history',       tabType: 'history',       singleton: true,  defaultTitle: '历史记录' },
  { routeName: 'settings',      tabType: 'settings',      singleton: true,  defaultTitle: '设置' },
  { routeName: 'recycleBin',    tabType: 'recycleBin',    singleton: true,  defaultTitle: '回收站' },
]

/** 按 routeName 查找配置 */
export function getTabRouteConfig(routeName: string): TabRouteConfig | undefined {
  return TAB_ROUTE_CONFIG.find(c => c.routeName === routeName)
}
```

### 任务 3: 重构 tabsStore.addTab 使用独立配置表

**文件：**
- 修改：`packages/webui/src/stores/tabs.ts`

**规格引用：** behavior-spec.md — `addTabByRoute` 逻辑

- [ ] **步骤 1: 使用 `getTabRouteConfig` 替代 `router.getRoutes()`**

```typescript
// 变更：import 独立配置表而非 router 实例（避免循环依赖）
import { getTabRouteConfig } from '@/router/tab-routes'

function addTabByRoute(routeName: string): Tab | null {
  const config = getTabRouteConfig(routeName)
  if (!config) {
    console.warn(`[tabsStore] No tab config for route "${routeName}", skipping`)
    return null
  }

  // 单例检查
  if (config.singleton) {
    const existing = tabs.value.find(t => t.type === config.tabType)
    if (existing) {
      activeTabId.value = existing.id
      return existing
    }
  }

  const id = config.singleton ? `tab-${config.tabType}` : `tab-${Date.now()}`
  const newTab: Tab = {
    id,
    type: config.tabType,
    title: config.defaultTitle,
    closable: true,
  }
  tabs.value.push(newTab)
  activeTabId.value = id
  return newTab
}
```

- [ ] **步骤 2: 移除 f-16 中硬编码的 SINGLETON_TYPES 和 getDefaultTitle**

删除 `SINGLETON_TYPES` Set 和 `getDefaultTitle` 函数，改为由 `getTabRouteConfig` 提供。

- [ ] **步骤 3: 保留原有 addTab 作为内部方法**

```typescript
// 直接添加方法，不做 singleton 检查（addTabByRoute 统一入口处理）
function addTabDirect(type: TabType, sessionId?: string, title?: string): Tab {
  const id = type === 'chat' ? `tab-${Date.now()}` : `tab-${type}`
  const newTab: Tab = {
    id,
    type,
    title: title ?? getDefaultTitle(type),
    sessionId,
    closable: true,

  return newTab
```
