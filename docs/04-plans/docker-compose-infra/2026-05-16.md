# Docker Compose 开发基础设施 实现计划

> **For agentic workers:** 必需子技能：superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans。步骤使用复选框（`- [ ]`）语法追踪。

**目标：** 搭建 Docker Compose 开发环境，包含 PostgreSQL、MinIO、Milvus、Redis 四个核心服务，支持 `pnpm infra:up` 一键启动和 `pnpm infra:down` 一键停止。

**架构：** 使用 `docker-compose.dev.yml` 定义四个基础设施服务容器，数据卷挂载到 `./.data/{service}` 实现开发环境持久化。Server（Hono Gateway）不在 Docker 中运行，保持独立进程以便热重载和调试。通过根目录 `package.json` 脚本封装 `docker compose` 命令。

**技术栈：** Docker Compose、PostgreSQL 16、MinIO、Milvus 2.4+ standalone、Redis 7

**Issue 引用：** [docs/02-issues/i-01-docker-compose-infra.md](../../02-issues/i-01-docker-compose-infra.md)
**Spec 引用：** [docs/03-specs/features/docker-compose-infra/](../../03-specs/features/docker-compose-infra/)
**测试用例：** 无（基础设施配置，通过健康检查命令验证）

### 任务 2: 创建 `docker-compose.dev.yml`

**文件：**
- 创建：`docker-compose.dev.yml`

**规格引用：**
- 功能规格：[范围内 — `docker-compose.dev.yml` 定义四个服务]
- 行为规格：[服务启动流程、健康检查流程、数据持久化策略]
- PRD：[2.3 Docker 基础设施]

- [ ] **步骤 1: 编写 `docker-compose.dev.yml`**

```yaml
name: goferbot-infra

services:
  postgres:
    container_name: goferbot-postgres
    image: postgres:16-alpine
    restart: unless-stopped
    ports:
      - "${POSTGRES_PORT:-5432}:5432"
    environment:
      POSTGRES_USER: ${POSTGRES_USER:-gofer}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-gofer_dev_pass}
      POSTGRES_DB: ${POSTGRES_DB:-goferbot}
    volumes:
      - ./.data/postgres:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U $${POSTGRES_USER} -d $${POSTGRES_DB}"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s
    networks:
      - goferbot-infra

  minio:
    container_name: goferbot-minio
    image: minio/minio:RELEASE.2025-01-20T14-49-07Z
    restart: unless-stopped
    command: server /data --console-address ":9001"
    ports:
      - "${MINIO_PORT:-9000}:9000"
      - "${MINIO_CONSOLE_PORT:-9001}:9001"
    environment:
      MINIO_ROOT_USER: ${MINIO_ROOT_USER:-minioadmin}
      MINIO_ROOT_PASSWORD: ${MINIO_ROOT_PASSWORD:-minioadmin}
    volumes:
      - ./.data/minio:/data
    healthcheck:
      test: ["CMD-SHELL", "curl -f http://localhost:9000/minio/health/live"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s
    networks:
      - goferbot-infra

  milvus:
    container_name: goferbot-milvus
    image: milvusdb/milvus:v2.4.17
    restart: unless-stopped
    command: ["milvus", "run", "standalone"]
    ports:
      - "${MILVUS_PORT:-19530}:19530"
      - "${MILVUS_METRIC_PORT:-9091}:9091"
    environment:
      ETCD_ENDPOINTS: etcd:2379
      MINIO_ADDRESS: minio:9000
    volumes:
      - ./.data/milvus:/var/lib/milvus
    healthcheck:
      test: ["CMD-SHELL", "curl -f http://localhost:9091/healthz"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 60s
    depends_on:
      - minio
    networks:
      - goferbot-infra

  redis:
    container_name: goferbot-redis
    image: redis:7-alpine
    restart: unless-stopped
    ports:
      - "${REDIS_PORT:-6379}:6379"
    command: redis-server --appendonly no
    volumes:
      - ./.data/redis:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 5s
    networks:
      - goferbot-infra

networks:
  goferbot-infra:
    driver: bridge
```

- [ ] **步骤 2: 验证 YAML 语法**

运行：`docker compose -f docker-compose.dev.yml config`
预期：成功解析并输出合并后的配置，无错误

### 任务 6: 启动服务并验证健康检查

**文件：**
- 无文件修改，纯验证

**规格引用：**
- Issue：[验收标准 — 启动后各服务健康检查通过]
- 行为规格：[健康检查流程、服务启动流程]

- [ ] **步骤 1: 启动所有服务**

运行：`pnpm infra:up`
预期：
- Docker Compose 创建网络 `goferbot-infra`
- 四个容器依次启动：`goferbot-postgres`、`goferbot-minio`、`goferbot-milvus`、`goferbot-redis`
- `--wait` 等待所有服务 healthcheck 通过后命令退出
- 终端输出服务状态摘要，退出码 0

- [ ] **步骤 2: 验证容器状态**

运行：`docker compose -f docker-compose.dev.yml ps`
预期：四个容器 `STATUS` 列均显示 `healthy`（或 `Up (healthy)`）

- [ ] **步骤 3: 验证 PostgreSQL 可连接**

运行：`docker exec goferbot-postgres pg_isready -U gofer -d goferbot`
预期：输出 `/var/run/postgresql:5432 - accepting connections`

- [ ] **步骤 4: 验证 MinIO 健康端点**

运行：`curl -f http://localhost:9000/minio/health/live`
预期：HTTP 200，无输出（或返回空响应），curl 退出码 0

- [ ] **步骤 5: 验证 MinIO 控制台可访问**

运行：`curl -s -o /dev/null -w "%{http_code}" http://localhost:9001`
预期：输出 `200` 或 `302`

- [ ] **步骤 6: 验证 Milvus 健康端点**

运行：`curl -f http://localhost:9091/healthz`
预期：HTTP 200，返回 `OK` 或空响应，curl 退出码 0

- [ ] **步骤 7: 验证 Milvus gRPC 端口监听**

运行：`docker exec goferbot-milvus netstat -tlnp | grep 19530`（或 `ss -tlnp | grep 19530`）
预期：显示 `19530` 端口处于 `LISTEN` 状态

- [ ] **步骤 8: 验证 Redis 可 ping**

运行：`docker exec goferbot-redis redis-cli ping`
预期：输出 `PONG`

- [ ] **步骤 9: 停止所有服务**

运行：`pnpm infra:down`
预期：
- 四个容器依次停止
- 网络保留或移除（取决于 Docker Compose 版本）
- 命令退出码 0

- [ ] **步骤 10: 验证容器已停止**

运行：`docker compose -f docker-compose.dev.yml ps`
预期：无运行中容器，或状态为 `Exited`

- [ ] **步骤 11: 验证数据卷保留**

运行：`ls -la .data/`
预期：显示 `postgres`、`minio`、`milvus`、`redis` 四个子目录，数据文件存在

- [ ] **步骤 12: 提交验证结果记录**

```bash
git commit --allow-empty -m "test(infra): verify docker-compose.dev.yml health checks pass"
```
