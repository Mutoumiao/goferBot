# q-01 NestJS 安全基线加固计划 v2

> **v1 废弃原因**：v1 基于 Hono 架构（hono-rate-limiter、hono/cors），当前服务端已全面迁移至 NestJS 10 + Fastify，安全基础设施均已用 NestJS 原生机制实现。v2 仅覆盖 NestJS 环境下的安全缺口。

## 现有安全措施（无需改动）

| 措施 | 实现方式 |
|------|---------|
| Helmet 安全头 | `@fastify/helmet`（bootstrap.ts） |
| CORS 白名单 | `@fastify/cors`（bootstrap.ts） |
| 速率限制 | `@nestjs/throttler` — default 60/min，auth 5/min 已定义（app.module.ts） |
| 输入校验 | `ZodValidationPipe`（APP_PIPE 全局注册） |
| 异常脱敏 | `AllExceptionsFilter`（生产环境隐藏堆栈/details） |
| JWT 认证 | `JwtAuthGuard` + 双令牌机制（15min/7d） |
| Chat SSRF | ChatDto 内置 baseUrl 白名单校验 |
| 爬虫拦截 | `SpiderGuard`（生产环境 UA 过滤） |
| 文件上传限制 | `@fastify/multipart` — 50MB/10 文件 |
| API Key 加密 | AES-256-GCM（settings.service.ts） |
