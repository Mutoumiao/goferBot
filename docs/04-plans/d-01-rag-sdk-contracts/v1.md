# RAG SDK 接口层实现计划

> **For agentic workers:** 必需子技能：superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans。步骤使用复选框（`- [ ]`）语法追踪。

**目标：** 为 `packages/rag-sdk/` 定义接口合约，结束当前空壳状态，使 Phase 5 的 RAG 实现只需遵循接口而无需修改消费方代码。

**架构：** 接口层位于 `packages/rag-sdk/src/interfaces/`，包含四个核心接口（IChunker / IEmbedder / IRetriever / IIndexer）、共享类型、错误类型和统一导出。SDK 不直接依赖 Milvus 客户端，仅通过 `IVectorStore` 类型引用实现协作。

**技术栈：** TypeScript 5.9、ESNext 模块、`bundler` 模块解析（与现有 `tsconfig.json` 一致）。

**Issue 引用：** `docs/02-issues/d-01-rag-sdk-contracts.md`
**Spec 引用：** `docs/03-specs/d-01-rag-sdk-contracts/`
**测试用例：** `docs/08-test-cases/d-01-rag-sdk-contracts.md`（预留，本计划不创建）

## 文件结构规划

```
packages/rag-sdk/src/
├── interfaces/
│   ├── IChunker.ts    # 文档分块策略抽象
│   ├── IEmbedder.ts   # 文本向量化抽象
│   ├── IRetriever.ts  # 语义检索抽象
│   ├── IIndexer.ts    # 向量索引写入抽象
│   └── index.ts       # 接口与类型的统一导出（barrel file）
├── types.ts           # 共享类型（DocumentSource / Chunk / ChunkWithScore / EmbeddingConfig / HybridSearchOptions）
├── errors.ts          # 错误类型体系（RAGError / EmbeddingError / RetrievalError / ValidationError）
└── index.ts           # 包入口：重新导出 interfaces、types、errors 全部公开成员
```

## 任务列表

### Task 1：创建共享类型 `types.ts`

- **文件**：创建 `packages/rag-sdk/src/types.ts`
- **规格引用**：API Spec 第 2 节「配置类型」、第 3 节「数据类型」；Behavior Spec 第 1.1 节「分块输入/输出」、第 2.1 节「向量化输入/输出」、第 3.1 节「检索输入/输出」
- **步骤**：
  1. 定义 `DocumentSource` 接口，字段：`documentId: string`、`kbId: string`、`content: string`、`mimeType: string`。
  2. 定义 `Chunk` 接口，字段：`id: string`、`documentId: string`、`kbId: string`、`content: string`、`chunkIndex: number`、`tokenCount?: number`。
  3. 定义 `ChunkWithScore` 接口，继承 `Chunk`，增加 `score: number`。
  4. 定义 `EmbeddingConfig` 接口，字段：`provider: string`、`model: string`、`dimension: number`、`apiKey: string`、`baseUrl?: string`。
  5. 定义 `HybridSearchOptions` 接口，字段：`vectorWeight?: number`、`keywordWeight?: number`、`rrfK?: number`。
  6. 为每个接口编写 JSDoc，说明用途和字段含义。
- **代码验证**：
  ```bash
  pnpm -F @goferbot/rag-sdk type-check
  ```

### Task 2：创建错误类型体系 `errors.ts`

- **文件**：创建 `packages/rag-sdk/src/errors.ts`
- **规格引用**：API Spec 第 4 节「错误类型」；Behavior Spec 第 5 节「错误场景」
- **步骤**：
  1. 定义 `RAGError` 基类，继承 `Error`，构造函数接收 `message: string` 和可选 `cause?: unknown`，显式设置 `this.name = 'RAGError'`。
  2. 定义 `EmbeddingError` 子类，继承 `RAGError`，构造函数接收 `message: string` 和可选 `cause?: unknown`，显式设置 `this.name = 'EmbeddingError'`。
  3. 定义 `RetrievalError` 子类，继承 `RAGError`，构造函数接收 `message: string` 和可选 `cause?: unknown`，显式设置 `this.name = 'RetrievalError'`。
  4. 定义 `ValidationError` 子类，继承 `RAGError`，构造函数接收 `message: string`（无 cause，与 API Spec 一致），显式设置 `this.name = 'ValidationError'`。
  5. 为每个错误类编写 JSDoc，说明触发场景。
- **代码验证**：
  ```bash
  pnpm -F @goferbot/rag-sdk type-check
  ```
