# 执行计划：i-00-core-interfaces

> 日期：2026-05-16
> 目标：定义四个核心接口抽象层（IRepository / IStorageProvider / IVectorStore / IAuthProvider），确保 V1→V2 迁移不重蹈紧耦合覆辙

## 文件结构规划

```
packages/server/src/interfaces/
├── errors.ts          # 错误类型体系（RepositoryError / NotFoundError / ConflictError / ValidationError / StorageError / VectorStoreError / AuthError）

├── IRepository.ts     # 泛型数据访问接口 + RepositoryFindOptions

├── IAuthProvider.ts   # 认证接口 + SignUpCredentials + SignInCredentials + AuthResult + User + Session
└── index.ts           # 统一导出（barrel file）
```

## 任务列表

### Task 1：创建错误类型体系 `errors.ts`

- **文件**：创建 `packages/server/src/interfaces/errors.ts`
- **规格引用**：API Spec 第 6 节「错误类型」、Behavior Spec 第 2 节「错误处理模式」
- **步骤**：
  1. 定义 `RepositoryError` 基类，继承 `Error`，支持可选 `cause` 参数。
  2. 定义六个业务错误子类：`NotFoundError`、`ConflictError`、`ValidationError`、`StorageError`、`VectorStoreError`、`AuthError`。
  3. `NotFoundError` 构造函数接收 `resource` 和 `id`，消息格式为 `${resource} ID ${id} 不存在`。
  4. 其余错误类接收 `message` 和可选 `cause`。
  5. 所有子类显式设置 `this.name`，确保 `instanceof` 和 `err.name` 判断可靠。
- **代码验证**：
  ```bash
  pnpm -F @goferbot/server type-check
  ```

### Task 3：创建泛型数据访问接口 `IRepository.ts`

- **文件**：创建 `packages/server/src/interfaces/IRepository.ts`
- **规格引用**：API Spec 第 1 节「IRepository<T>」、Behavior Spec 第 1 节「接口调用约定」、PRD 7.1 数据模型
- **步骤**：
  1. 从 `./types` 导入 `EntityBase`，从 `./errors` 导入错误类型（用于 JSDoc `@throws` 引用）。
  2. 定义 `RepositoryFindOptions` 接口，字段：`limit?: number`、`offset?: number`、`orderBy?: string`。
  3. 定义 `IRepository<T extends { id: string }>` 接口，包含五个方法：
     - `findById(id: string): Promise<T>` — `@throws NotFoundError`
     - `findAll(options?: RepositoryFindOptions): Promise<T[]>` — 空数组不抛异常
     - `create(data: Omit<T, 'id' | 'createdAt' | 'updatedAt'>): Promise<T>` — `@throws ConflictError | ValidationError`
     - `update(id: string, data: Partial<Omit<T, 'id' | 'createdAt'>>): Promise<T>` — `@throws NotFoundError`
     - `delete(id: string): Promise<void>` — `@throws NotFoundError`（实现也可静默成功）
  4. 为每个方法编写 JSDoc，标注参数、返回值、异常。
- **代码验证**：
  ```bash
  pnpm -F @goferbot/server type-check
  ```

### Task 4：创建文件存储接口 `IStorageProvider.ts`

- **文件**：创建 `packages/server/src/interfaces/IStorageProvider.ts`
- **规格引用**：API Spec 第 2 节「IStorageProvider」、Behavior Spec 第 1.4 节「流式与缓冲」、PRD 7.2 MinIO Storage Key 设计
- **步骤**：
  1. 从 `./errors` 导入 `StorageError`、`NotFoundError`。
  2. 定义 `StorageMeta` 接口，字段：`key: string`、`size: number`、`etag?: string`、`contentType: string`、`lastModified?: Date`。
  3. 定义 `StorageUrlOptions` 接口，字段：`expiresIn?: number`（默认 3600）。
  4. 定义 `IStorageProvider` 接口，包含四个方法：
     - `upload(key, stream, contentType): Promise<StorageMeta>` — stream 类型为 `ReadableStream<Uint8Array> | NodeJS.ReadableStream`
     - `download(key): Promise<ReadableStream<Uint8Array>>` — 返回可读流，非 Buffer
     - `delete(key): Promise<void>`
     - `getUrl(key, options?): Promise<string>`
  5. 为每个方法编写 JSDoc，标注 `@throws`。
- **代码验证**：
  ```bash
  pnpm -F @goferbot/server type-check
  ```

### Task 5：创建向量存储接口 `IVectorStore.ts`

- **文件**：创建 `packages/server/src/interfaces/IVectorStore.ts`
- **规格引用**：API Spec 第 3 节「IVectorStore」、Behavior Spec 第 3.1 节「初始化时序」、PRD 7.3 Milvus Collection 设计
- **步骤**：
  1. 从 `./errors` 导入 `VectorStoreError`。
  2. 定义 `VectorRecord` 接口，字段：`id: string`（Milvus 主键）、`chunkId: string`（关联 chunks.id）、`kbId: string`、`fileId: string`、`embedding: number[]`（维度 1536）。
  3. 定义 `VectorSearchOptions` 接口，字段：`topK?: number`、`filter?: { kbId?: string }`。
  4. 定义 `VectorSearchResult` 接口，字段：`id: string`、`chunkId: string`、`score: number`。
  5. 定义 `IVectorStore` 接口，包含四个方法：
     - `insertVectors(vectors: VectorRecord[]): Promise<void>`
     - `searchVectors(queryVector: number[], options?: VectorSearchOptions): Promise<VectorSearchResult[]>`
     - `deleteByIds(ids: string[]): Promise<void>`
     - `ensureCollection(): Promise<void>` — 幂等，启动时调用一次
  6. 为每个方法编写 JSDoc，标注 `@throws`。
- **代码验证**：
  ```bash
  pnpm -F @goferbot/server type-check
  ```

### Task 6：创建认证接口 `IAuthProvider.ts`

- **文件**：创建 `packages/server/src/interfaces/IAuthProvider.ts`
- **规格引用**：API Spec 第 4 节「IAuthProvider」、Behavior Spec 第 3.3 节「会话与上下文」、PRD 3.1 认证系统
- **步骤**：
  1. 从 `hono` 导入 `MiddlewareHandler` 类型（依赖已存在于 `package.json`）。
  2. 从 `./errors` 导入 `ConflictError`、`ValidationError`、`AuthError`。
  3. 定义 `SignUpCredentials`（`email`、`password`、`name?`）、`SignInCredentials`（`email`、`password`）。
  4. 定义 `User`（`id`、`email`、`name | null`、`avatar | null`、`createdAt: Date`）。
  5. 定义 `Session`（`id`、`userId`、`expiresAt: Date`）。
  6. 定义 `AuthResult`（`user: User`、`session: Session`）。
  7. 定义 `IAuthProvider` 接口，包含五个方法：
     - `signUp(credentials): Promise<AuthResult>` — `@throws ConflictError | ValidationError`
     - `signIn(credentials): Promise<AuthResult>` — `@throws AuthError`
     - `signOut(request: Request): Promise<void>`
     - `getSession(request: Request): Promise<Session | null>` — 未登录返回 `null`，不抛异常
     - `middleware(): MiddlewareHandler` — 内部调用 `getSession`，未登录返回 401
  8. 为每个方法编写 JSDoc。
- **代码验证**：
  ```bash
  pnpm -F @goferbot/server type-check
  ```
