# API 规格模板（后端）

```markdown
# API 规格：{功能名称}

## 端点

### POST /api/{资源}

#### 认证
{Bearer Token / 无 / 等}

#### 请求
```json
{
  "field": "type (constraints)"
}
```

#### 响应 200/201
```json
{
  "field": "type"
}
```

#### 错误码
| 码 | 场景 | 响应体 |
|------|----------|---------------|
| 400 | {场景} | `{ "error": "..." }` |
| 409 | {场景} | `{ "error": "..." }` |

#### 异步行为
- {异步发生什么}
- {客户端如何轮询状态}
```

---

## 关键规则

- 每个端点必须有请求/响应示例
- 每个错误码必须有触发场景
- 异步行为必须说明客户端如何获取状态更新
