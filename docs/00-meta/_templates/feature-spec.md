# 功能规格模板

```markdown
# 功能规格：{功能名称}

## 用户故事

作为 {用户类型}，我希望 {操作}，以便 {收益}。

## 边界

- 范围内：{包含的内容}
- 范围外：{明确排除的内容}

## 涉及页面/组件

- {页面/组件 1}
- {页面/组件 2}

## 相关功能

- {上游功能} — 提供 {什么}
- {下游功能} — 消费 {什么}

## 已做决策

| 决策 | 理由 | 可逆？ |
|----------|-----------|-------------|
| {决策 1} | {原因} | {是/否} |
```

---

## 填写说明

| 字段 | 说明 |
|------|------|
| `用户故事` | 标准三段式，明确用户、操作、收益 |
| `边界` | 范围外必须明确写出"本期不做"的内容 |
| `涉及页面/组件` | 列出所有需要新建或修改的文件 |
| `相关功能` | 建立功能依赖图，标明数据流向 |
| `已做决策` | 记录不可撤销或高成本的决策 |
