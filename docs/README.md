# 文档导航

> Issue-Centric 结构：一个 issue 一个目录，所有文档集中在 `docs/issues/` 下。

| 目录                        | 内容                             | 何时阅读             |
|-----------------------------|----------------------------------|----------------------|
| [meta](guide/)              | 文档规范、开发流程               | 新成员入职、写文档前 |
| [01-prd](01-prd/)           | 产品需求文档                     | 规划阶段             |
| [issues](issues/)           | 活跃 Issue（Issue-Centric 结构） | 领取任务时           |
| [adrs](adrs/)               | 架构决策记录                     | 做架构决策前         |
| [design](design/)           | 设计系统、视觉稿                 | 实现 UI 前           |
| [reviews](reviews/)         | 审查记录                         | 修改已有功能前       |
| [99-archived](99-archived/) | 历史归档                         | 考古时               |

## 快速开始

1. 读 [guide/workflow.md](guide/workflow.md) 了解开发流程
2. 读 [01-prd/v2-cloud-native.md](01-prd/v2-cloud-native.md) 了解产品愿景
3. 查 [BACKLOG.md](../BACKLOG.md) 或 [CHANGELOG.md](../CHANGELOG.md) 定位目标 issue
4. 到 [issues/](issues/) 阅读对应 issue 的 `issue.md` 和 `specs/`
