# core 模块设计文档

> 共享领域契约层，定义 RAG 系统的跨模块数据模型、校验规则与能力接口。
> **注意**：当前采用扁平化设计，core 内容直接位于 `src/` 根目录，不再单独设 `core/` 子目录。

---

## 1. 定位

core 是 `@goferbot/rag-sdk` 的基础契约层，所有其他模块（indexing / runtime / observability）均依赖它。

### 1.1 职责边界

**core 负责：**
- 共享数据模型（DocumentSource / Chunk / ChunkWithScore / EmbeddingConfig / HybridSearchOptions / Query / RetrievalCandidate）
- 数据校验规则（Zod Schema）
- 跨模块能力接口（IChunker / IEmbedder / IIndexer / IRetriever / IReranker / IGenerator / IVectorStore）
- 通用错误模型（RAGError 体系）
- Pipeline 类型级定义

**core 不负责：**
- 业务实现
- 外部 SDK 调用
- 模块内部私有结构

### 1.2 核心原则

1. core 是"跨模块共享契约"的唯一来源。
2. 模块内部私有结构允许存在，但不得与 core 类型冲突。
3. 共享数据模型不得在其他模块重复定义。

```
src/
  ├── types.ts          # 类型定义（现有接口迁移 + 扩展）

  ├── vector-store.ts   # 向量存储接口（新增，解耦 server）

```

### 强约束

1. 所有契约文件必须在 `src/` 根目录下。
2. 不允许在子模块内重复定义 core 类型。
3. 所有导出统一从 `src/index.ts` 暴露。

## 3. 数据模型

### 3.1 基础类型

```typescript
// src/types.ts

/**
 * 文档源输入。
 * 表示待分块的原始文档，由解析器（Parser）产出后传入 IChunker。
 */
export interface DocumentSource {
  /** 文档在 PostgreSQL 中的持久化 ID */
  documentId: string
  /** 所属知识库 ID */
  kbId: string
  /** 文档纯文本内容（已提取） */
  content: string
  /** MIME 类型，如 `text/plain`、`application/pdf` */
  mimeType: string
}

/**
 * 查询对象。
 * 不是简单字符串，而是结构化的检索请求，支持重写、扩展、过滤。
 */
export interface Query {
  /** 用户原始输入 */
  original: string
  /** 重写后的查询（可选） */
  rewritten?: string
  /** 扩展查询词（可选） */
  expanded?: string[]
  /** 目标知识库 ID 列表 */
  kbIds: string[]
  /** 附加过滤条件（可选） */
  filters?: Record<string, unknown>
}

/**
 * 文本块。
 * IChunker 的输出单元，每个 Chunk 对应原始文档中的一个语义片段。
 */
export interface Chunk {
  /** 块全局唯一 ID */
  id: string
  /** 来源文档 ID */
  documentId: string
  /** 所属知识库 ID */
  kbId: string
  /** 块纯文本内容 */
  content: string
  /** 块在文档内的递增序号，从 0 开始 */
  chunkIndex: number
  /** 预估 token 数（可选） */
  tokenCount?: number
  /** 父 chunk ID（用于 Small-to-Big Retrieval） */
  parentId?: string
  /** 层级路径（如 ['第一章', '第一节']） */
  hierarchyPath?: string[]
  /** 附加元数据 */
  metadata?: {
    /** 标题链 */
    headers?: string[]
    /** 所属章节 */
    section?: string
    /** 其他自定义字段 */
    [key: string]: unknown
  }
}

/**
 * 带相关度分数的文本块。
 * IRetriever 的检索结果单元。
 */
export interface ChunkWithScore extends Chunk {
  /** 相关度分数，越高越相关 */
  score: number
}

/**
 * 检索候选。
 * 包含 chunk、分数、来源等完整信息，供 post-retrieval 处理。
 */
export interface RetrievalCandidate {
  /** 候选 chunk */
  chunk: Chunk
  /** 相关度分数 */
  score: number
  /** 检索来源 */
  source: 'vector' | 'keyword' | 'hybrid'
  /** 检索路由信息（如使用了哪个索引） */
  route?: string
  /** 附加元数据 */
  metadata?: Record<string, unknown>
}

/**
 * 向量化配置。
 */
export interface EmbeddingConfig {
  /** 提供商标识，如 `openai`、`cohere`、`local` */
  provider: string
  /** 模型名称，如 `text-embedding-3-small` */
  model: string
  /** 输出向量维度 */
  dimension: number
  /** API 密钥 */
  apiKey: string
  /** 自定义 API 基础地址（可选） */
  baseUrl?: string
}

/**
 * 混合检索参数（预留）。
 */
export interface HybridSearchOptions {
  /** RRF 融合参数 k，默认 60 */
  rrfK?: number
  /** 向量检索权重，范围 0~1，默认 0.5（预留，当前版本使用 RRF） */
  vectorWeight?: number
  /** 关键词检索权重，范围 0~1，默认 0.5（预留，当前版本使用 RRF） */
  keywordWeight?: number
}
```

### 3.2 Zod Schema

```typescript
// src/schema.ts
import { z } from 'zod'

export const DocumentSourceSchema = z.object({
  documentId: z.string().uuid(),
  kbId: z.string().uuid(),
  content: z.string().min(1),
  mimeType: z.string().min(1),
})

export const QuerySchema = z.object({
  original: z.string().min(1),
  rewritten: z.string().optional(),
  expanded: z.array(z.string()).optional(),
  kbIds: z.array(z.string().uuid()).min(1),
  filters: z.record(z.unknown()).optional(),
})

export const ChunkSchema = z.object({
  id: z.string().uuid(),
  documentId: z.string().uuid(),
  kbId: z.string().uuid(),
  content: z.string().min(1),
  chunkIndex: z.number().int().min(0),
  tokenCount: z.number().int().optional(),
  parentId: z.string().uuid().optional(),
  hierarchyPath: z.array(z.string()).optional(),
  metadata: z.record(z.unknown()).optional(),
})

export const ChunkWithScoreSchema = ChunkSchema.extend({
  score: z.number().min(0).max(1),
})

export const RetrievalCandidateSchema = z.object({
  chunk: ChunkSchema,
  score: z.number().min(0).max(1),
  source: z.enum(['vector', 'keyword', 'hybrid']),
  route: z.string().optional(),
  metadata: z.record(z.unknown()).optional(),
})

export const EmbeddingConfigSchema = z.object({
  provider: z.string().min(1),
  model: z.string().min(1),
  dimension: z.number().int().positive(),
  apiKey: z.string(),
  baseUrl: z.string().url().optional(),
})

export const HybridSearchOptionsSchema = z.object({
  vectorWeight: z.number().min(0).max(1).optional(),
  keywordWeight: z.number().min(0).max(1).optional(),
  rrfK: z.number().int().positive().optional(),
})
```

### 3.3 数据模型规则

1. 核心数据模型必须由 Zod 定义。
2. 类型必须由 `z.infer` 推导。
3. metadata 必须是 JSON-safe 结构。
4. 非核心类型允许手写。

## 4. 接口设计

### 4.1 IChunker

```typescript
// src/interfaces.ts

export interface IChunker {
  /**
   * 将单个文档切分为文本块数组。
   * @param doc - 文档源输入
   * @returns 按 chunkIndex 递增排序的 Chunk 数组
   * @throws ValidationError 当输入内容格式非法或分块参数冲突时
   */
  chunk(doc: DocumentSource): Promise<Chunk[]>
}
```

### 4.2 IEmbedder

```typescript
export interface IEmbedder {
  /**
   * 将文本数组批量转换为向量数组。
   * @param texts - 待嵌入的纯文本数组，长度 >= 1
   * @returns 与输入一一对应的高维向量数组
   * @throws ValidationError 当 texts 为空数组时
   * @throws EmbeddingError 当嵌入 API 失败或返回维度不匹配时
   */
  embed(texts: string[]): Promise<number[][]>
  /** 当前 Embedder 的配置信息（只读） */
  readonly config: Readonly<EmbeddingConfig>
}
```

### 4.3 IIndexer

```typescript
export interface IIndexer {
  /**
   * 将文本块及其向量批量写入向量索引。
   * @param chunks - 已持久化到 PostgreSQL 的 Chunk 数组
   * @param vectors - 与 chunks 一一对应的嵌入向量数组
   * @throws ValidationError 当 chunks 与 vectors 长度不一致时
   * @throws EmbeddingError 当向量维度与配置不匹配时
   * @throws RAGError 当向量库写入失败时
   */
  index(chunks: Chunk[], vectors: number[][]): Promise<void>
}
```

### 4.4 IRetriever（增强版）

```typescript
export interface IRetriever {
  /**
   * 执行混合检索（向量 + 关键词 + RRF 融合）。
   * @param query - 结构化查询对象（非简单字符串）
   * @param topK - 返回结果数量上限，默认 5
   * @param options - 混合检索参数
   * @returns 按 RRF 融合分数降序排列的 RetrievalCandidate 数组
   * @throws ValidationError 当 query.original 为空或 query.kbIds 为空数组时
   * @throws RetrievalError 当检索执行失败时
   */
  retrieve(
    query: Query,
    topK?: number,
    options?: HybridSearchOptions,
  ): Promise<RetrievalCandidate[]>
}
```

**关键变更**：
- 入参从 `query: string, kbIds: string[]` 改为 `query: Query`
- 返回值从 `ChunkWithScore[]` 改为 `RetrievalCandidate[]`
- 支持 `source`、`route`、`metadata` 等扩展信息

### 4.5 IReranker（新增）

```typescript
export interface IReranker {
  /**
   * 对检索候选进行重排序。
   * @param candidates - 初始检索结果
   * @param query - 原始查询
   * @returns 按重排分数降序排列的 RetrievalCandidate 数组
   */
  rerank(candidates: RetrievalCandidate[], query: Query): Promise<RetrievalCandidate[]>
}
```

### 4.6 IGenerator

```typescript
export interface IGenerator {
  /**
   * 基于查询和上下文生成回答。
   * @param input.query - 查询对象
   * @param input.chunks - 选中的上下文 chunks
   * @returns 生成的回答文本
```

### 4.7 IVectorStore（新增，解耦 server）

```typescript
// src/vector-store.ts

export interface IVectorStore {
  /** 批量插入向量记录 */
  insertVectors(vectors: VectorRecord[]): Promise<void>
  /** ANN 近似最近邻搜索 */
  searchVectors(
    queryVector: number[],
    options?: VectorSearchOptions,
  ): Promise<VectorSearchResult[]>
  /** 根据 ID 删除向量 */
  deleteByIds(ids: string[]): Promise<void>
  /** 幂等地创建/校验 collection 结构 */
  ensureCollection(): Promise<void>
}

export interface VectorRecord {
  /** 向量记录主键 */
  id: string
  /** 关联 PostgreSQL chunks.id */
  chunkId: string
  /** 知识库 ID，用于过滤 */
  kbId: string
  /** 文档 ID */
  fileId: string
  /** 向量数据 */
  embedding: number[]
}

export interface VectorSearchOptions {
  /** 返回结果数量，默认 5 */
  topK?: number
  /** 过滤条件，当前仅支持 kbId 精确匹配 */
  filter?: {
    kbId?: string
  }
}

export interface VectorSearchResult {
  /** 向量记录主键 */
  id: string
  /** 关联 chunkId */
  chunkId: string
  /** 相似度分数（0~1，越高越相似） */
  score: number
}
```

**设计说明**：
- 将 server 的 `IVectorStore` 完整复制到 SDK 内部
- server 的 `VectorService` 实现此接口，通过构造函数注入
- SDK 不再反向引用 server 的任何文件

## 10. 关键变更记录

| 变更项 | 原设计 | 新设计 | 原因 |
|--------|--------|--------|------|
| core 目录 | `src/core/` | 扁平化到 `src/` 根目录 | 单包内减少不必要的目录深度 |
| Query 类型 | 无，使用 `string` | 新增 `Query` 接口 | 支持 query rewrite、expansion、routing |
| Chunk 字段 | 基础字段 | 新增 `parentId`、`hierarchyPath`、`metadata` | 支持 Small-to-Big Retrieval、层级索引 |
| IRetriever 入参 | `query: string, kbIds: string[]` | `query: Query` | 统一查询抽象 |
| IRetriever 返回值 | `ChunkWithScore[]` | `RetrievalCandidate[]` | 支持 source、route、metadata |
| IReranker | 无 | 新增接口 | 支持检索后重排序 |
| IRetriever 职责 | 仅向量检索 | 混合检索（向量 + 关键词 + RRF） | 主流检索方案，提升召回率 |
| IVectorStore | 引用 server | 内聚到 `src/vector-store.ts` | 彻底解耦反向依赖 |
| IndexingResult | 仅 chunks + vectorCount | 新增 `stages` 字段 | 记录每阶段状态，支持可观测性 |
| RuntimeDebugInfo | 基础计数器 | 新增 `traceId`、`stages` | 支持 Trace 和阶段耗时 breakdown |
