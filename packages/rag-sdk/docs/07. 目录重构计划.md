# 目录重构计划

> 将现有 `packages/rag-sdk/src/` 从扁平结构重构为模块化的多目录结构。

---

## 1. 当前结构

```
packages/rag-sdk/src/
├── index.ts              # 导出 interfaces / types / errors
├── types.ts              # DocumentSource / Chunk / ChunkWithScore / EmbeddingConfig / HybridSearchOptions
├── errors.ts             # RAGError / EmbeddingError / RetrievalError / ValidationError
└── interfaces/
    ├── index.ts          # 导出 4 个接口
    ├── IChunker.ts
    ├── IEmbedder.ts
    ├── IIndexer.ts
    └── IRetriever.ts
```

**问题**：
- 无 Zod Schema 定义
- 无 Pipeline 抽象
- 无具体实现（chunkers / embedders / retrievers）
- 接口分散在多个文件
- 无 Query 抽象（使用简单字符串）
- 无 IVectorStore 接口（反向依赖 server）
- 无 observability 能力

---

## 4. 重构步骤

### Step 1：创建目录结构

```bash
cd packages/rag-sdk/src
mkdir -p indexing/chunkers indexing/embedders indexing/indexers
mkdir -p runtime/retrievers runtime/postprocessors runtime/utils
mkdir -p observability
```

### Step 2：扩展核心契约

1. 扩展 `types.ts`：新增 `Query`、`RetrievalCandidate`、`Chunk` 字段（parentId / hierarchyPath / metadata）
2. 新建 `schema.ts`：为所有类型定义 Zod Schema
3. 新建 `interfaces.ts`：合并 4 个接口，新增 `IReranker`、`IGenerator`、`IVectorStore`
4. 扩展 `errors.ts`：新增 `IndexingError`
5. 新建 `pipeline.ts`：定义 `IndexingPipeline`、`RuntimePipeline`、`IndexingStage`、`RuntimeStage`
6. 新建 `vector-store.ts`：定义 `IVectorStore`、`VectorRecord`、`VectorSearchOptions`
7. 新建 `interfaces.ts`：合并 4 个接口，新增 `IReranker`、`IGenerator`、`IVectorStore`、`IKeywordStore`

### Step 3：删除旧文件

```bash
rm -rf interfaces/
```

### Step 4：实现 indexing 模块

1. 实现 `RecursiveCharacterChunker`（支持 parentId / hierarchyPath）
2. 实现 `OpenAIEmbedder`（从原 adapters 迁移）
3. 实现 `MilvusIndexer`
4. 实现 `runIndexing`（支持阶段状态追踪）

### Step 5：实现 runtime 模块

1. 实现 `HybridRetriever`（支持向量检索 + 关键词检索 + RRF 融合）
2. 定义 `IKeywordStore` 接口（由 server 基于 PostgreSQL FTS 实现）
3. 实现 `reciprocalRankFusion` 算法
4. 实现 `DefaultRetrievalPostprocessor`（支持分数过滤、rerank、budget trim、trace）
5. 实现 `runRetrievalPipeline`（支持阶段耗时记录）

### Step 6：实现 observability 模块

1. 定义 `RAGTrace`、`RAGStage`、`RAGObserver` 类型
2. 实现 `RAGTracer` 和 `consoleObserver`

### Step 7：更新根入口

重写 `src/index.ts`，聚合所有模块导出：

```typescript
// src/index.ts
export * from './types.js'
export * from './schema.js'
export * from './interfaces.js'
export * from './errors.js'
export * from './pipeline.js'
export * from './vector-store.js'
export * from './indexing/index.js'
export * from './runtime/index.js'
export * from './observability/index.js'
```

### Step 8：验证

```bash
pnpm type-check
pnpm test
pnpm build
```

## 5. 风险与注意事项

1. **server 引用路径**：重构后 `packages/server` 中引用 `@goferbot/rag-sdk` 的方式不变（通过 npm workspace），但需确保 `pnpm build` 后产物路径正确。
2. **现有接口冻结**：`IChunker` / `IEmbedder` / `IIndexer` 的方法签名保持不变。`IRetriever` 签名变更（入参从 string 改为 Query，返回值从 ChunkWithScore[] 改为 RetrievalCandidate[]），需在 server 中同步更新调用方。
3. **Zod 引入**：新增 `zod` 依赖到 `packages/rag-sdk/package.json`。
4. **openai 依赖**：新增 `openai` 依赖到 `packages/rag-sdk/package.json`。
5. **循环依赖检查**：`runtime` 引用 core，`indexing` 引用 core，`observability` 引用 core，禁止反向依赖。
6. **server 同步更新**：server 中的 `IVectorStore` 接口需与 SDK 的 `IVectorStore` 保持一致，或直接使用 SDK 导出的接口。

## 6. 关键变更记录

| 维度 | 原设计 | 新设计 | 原因 |
|------|--------|--------|------|
| core 结构 | `src/core/` 子目录 | 扁平化到 `src/` 根目录 | 单包内减少不必要的目录深度 |
| adapters 模块 | 独立的 `src/adapters/` | **删除** | 当前仅 OpenAI 一种适配，直接放在 indexing/embedders/ |
| Query 抽象 | 无，使用 `string` | 新增 `Query` 接口 | 支持 query rewrite、expansion、routing |
| Chunk 字段 | 基础字段 | 新增 `parentId`、`hierarchyPath`、`metadata` | 支持 Small-to-Big Retrieval、层级索引 |
| IRetriever 入参 | `query: string, kbIds: string[]` | `query: Query` | 统一查询抽象 |
| IRetriever 返回值 | `ChunkWithScore[]` | `RetrievalCandidate[]` | 支持 source、route、metadata |
| 检索策略 | 仅向量检索（`MilvusRetriever`） | 混合检索（`HybridRetriever`：向量 + 关键词 + RRF） | 主流方案，提升召回率 |
| 关键词检索 | 无 | 新增 `IKeywordStore` 接口 | 支持 PostgreSQL Full-Text Search |
| RRF 融合 | 无 | 新增 `src/runtime/utils/rrf.ts` | 融合多路检索结果 |
| IVectorStore | 引用 server | 内聚到 `src/vector-store.ts` | 彻底解耦反向依赖 |
| IndexingResult | 仅 chunks + vectorCount | 新增 `stages` 字段 | 记录每阶段状态，支持可观测性 |
| RuntimeDebugInfo | 基础计数器 | 升级为 `RAGTrace` | 支持 traceId、stages、metrics |
| observability | 无 | 新增 `src/observability/` | 内置可观测性能力 |
| Post-Retrieval | 仅分数过滤 + 预算截断 | 分层处理：filter → rerank → budget trim | 支持更丰富的检索后处理能力 |
| SelectionTrace | 无 | 新增 | 记录每步操作原因，支持调试 |
