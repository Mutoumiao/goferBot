# observability 模块设计文档

> 可观测性模块，负责 RAG 链路的 Trace、Metrics 和调试信息收集。
> 对应原独立多包方案中的 `packages/observability`，但当前为精简版，内置于 SDK 核心流程。

---

## 1. 模块定位

`observability` 是 `@goferbot/rag-sdk` 内部的可观测性模块。RAG 不是简单函数调用，而是一条多阶段流水线，问题定位极度依赖中间过程。如果 SDK 只返回最终答案而不暴露中间发生了什么，开发者只能靠猜。

### 1.1 为什么需要观测系统

典型场景：用户反馈"回答不对"。可能原因：
- query rewrite 改坏了原意
- 检索阶段没召回到正确文档
- rerank/filter 把正确内容丢了
- prompt 里上下文不合适
- 某一步太慢导致超时

没有观测能力，开发者只能看到：
```typescript
const result = await runtime.run({ query: "公司年假政策是什么？" })
console.log(result.answer) // 不知道中间发生了什么
```

### 1.2 设计目标

1. 每次请求有唯一 traceId，串联所有事件
2. 每阶段输入输出、耗时、错误完整记录
3. 支持自定义 Observer 接入外部系统（如 LangSmith、Phoenix）
4. 对 SDK 用户透明，默认开启，可配置关闭

---

## 2. 职责边界

### 2.1 observability 负责什么

1. Trace 生命周期管理（start → stage → complete/error）
2. Metrics 收集（召回数、选中数、丢弃数、token 数、耗时）
3. 默认 ConsoleObserver 输出
4. Observer 接口定义，支持自定义 exporter

### 2.2 observability 不负责什么

1. 不定义共享数据模型（由 core 负责）
2. 不实现具体 exporter（如 OpenTelemetry、LangSmith 适配后续扩展）
3. 不存储历史数据（由外部系统负责）
4. 不做可视化 dashboard

## 4. 类型设计

```typescript
// src/observability/types.ts

export interface RAGTrace {
  /** 唯一追踪 ID */
  traceId: string
  /** 查询对象 */
  query: Query
  /** 各阶段记录 */
  stages: RAGStage[]
  /** 汇总指标 */
  metrics: RAGMetrics
  /** 开始时间 */
  startTime: number
  /** 结束时间 */
  endTime?: number
  /** 错误信息 */
  error?: string
}

export interface RAGStage {
  /** 阶段名称 */
  name: string
  /** 开始时间戳 */
  startTime: number
  /** 结束时间戳 */
  endTime: number
  /** 输入 */
  input: unknown
  /** 输出 */
  output: unknown
  /** 错误 */
  error?: string
}

export interface RAGMetrics {
  /** 召回数量 */
  retrievalCount: number
  /** 选中数量 */
  selectedCount: number
  /** 丢弃数量 */
  droppedCount: number
  /** 总 token 数 */
  totalTokens: number
  /** 总耗时 ms */
  latencyMs: number
}

export interface RAGObserver {
  /** 链路开始 */
  onStart?(trace: RAGTrace): void
  /** 阶段完成 */
  onStageComplete?(stage: string, trace: RAGTrace): void
  /** 链路完成 */
  onComplete?(trace: RAGTrace): void
  /** 链路出错 */
  onError?(error: Error, trace: RAGTrace): void
}
```

## 5. Tracer 实现

```typescript
// src/observability/tracer.ts
import type { RAGTrace, RAGStage, RAGObserver, RAGMetrics } from './types.js'
import type { Query } from '../types.js'

export class RAGTracer {
  private readonly observers: RAGObserver[]
  private trace?: RAGTrace

  constructor(observers: RAGObserver[] = []) {
    this.observers = observers
  }

  start(query: Query): void {
    this.trace = {
      traceId: crypto.randomUUID(),
      query,
      stages: [],
      metrics: {
        retrievalCount: 0,
        selectedCount: 0,
        droppedCount: 0,
        totalTokens: 0,
        latencyMs: 0,
      },
      startTime: Date.now(),
    }
    this.notify('onStart', this.trace)
  }

  stageStart(name: string, input: unknown): void {
    if (!this.trace) return
    this.trace.stages.push({
      name,
      startTime: Date.now(),
      endTime: 0,
      input,
      output: null,
    })
  }

  stageComplete(name: string, output: unknown): void {
    if (!this.trace) return
    const stage = this.trace.stages.find(s => s.name === name && s.endTime === 0)
    if (stage) {
      stage.endTime = Date.now()
      stage.output = output
      this.notify('onStageComplete', name, this.trace)
    }
  }

  stageError(name: string, error: Error): void {
    if (!this.trace) return
    const stage = this.trace.stages.find(s => s.name === name && s.endTime === 0)
    if (stage) {
      stage.endTime = Date.now()
      stage.error = error.message
    }
    this.notify('onError', error, this.trace)
  }

  complete(metrics: Partial<RAGMetrics>): void {
    if (!this.trace) return
    this.trace.endTime = Date.now()
    this.trace.metrics = {
      ...this.trace.metrics,
      ...metrics,
      latencyMs: this.trace.endTime - this.trace.startTime,
    }
    this.notify('onComplete', this.trace)
  }

  error(err: Error): void {
    if (!this.trace) return
    this.trace.endTime = Date.now()
    this.trace.error = err.message
    this.notify('onError', err, this.trace)
  }

  getTrace(): RAGTrace | undefined {
    return this.trace
  }

  private notify(event: 'onStart', trace: RAGTrace): void
  private notify(event: 'onStageComplete', stage: string, trace: RAGTrace): void
  private notify(event: 'onComplete', trace: RAGTrace): void
  private notify(event: 'onError', error: Error, trace: RAGTrace): void
  private notify(event: string, ...args: unknown[]): void {
    for (const observer of this.observers) {
      const handler = observer[event as keyof RAGObserver]
      if (handler) {
        try {
          (handler as Function).apply(observer, args)
        } catch {
          // Observer 错误不应影响主流程
        }
      }
    }
  }
}

/** 默认 ConsoleObserver，输出到控制台 */
export const consoleObserver: RAGObserver = {
  onStart(trace) {
    console.log(`[RAG Trace ${trace.traceId}] 开始: ${trace.query.original}`)
  },
  onStageComplete(stage, trace) {
    const s = trace.stages.find(st => st.name === stage)
    if (s) {
      console.log(`[RAG Trace ${trace.traceId}] 阶段完成: ${stage} (${s.endTime - s.startTime}ms)`)
    }
  },
  onComplete(trace) {
    console.log(`[RAG Trace ${trace.traceId}] 完成: 耗时 ${trace.metrics.latencyMs}ms, 召回 ${trace.metrics.retrievalCount}, 选中 ${trace.metrics.selectedCount}`)
  },
  onError(error, trace) {
    console.error(`[RAG Trace ${trace.traceId}] 错误: ${error.message}`)
  },
}
```

## 6. 与 Pipeline 集成

```typescript
// 在 runtime/pipeline.ts 中使用 Tracer

import { RAGTracer, consoleObserver } from '../observability/index.js'

export async function runRetrievalPipeline(
  query: Query,
  options: RuntimePipelineOptions
): Promise<RuntimePipelineResult> {
  const tracer = new RAGTracer([consoleObserver, ...(options.observers ?? [])])
  tracer.start(query)

  try {
    // Stage 1: Retrieval
    tracer.stageStart('retrieval', query)
    const candidates = await options.retriever.retrieve(query)
    tracer.stageComplete('retrieval', { count: candidates.length })

    // Stage 2: Post-Retrieval
    tracer.stageStart('post-retrieval', candidates)
    let selectedChunks: Chunk[]
    let droppedCount = 0
    if (options.postprocessor) {
      const result = await options.postprocessor.process(candidates, query)
      selectedChunks = result.selectedCandidates.map(c => c.chunk)
      droppedCount = result.droppedCandidates.length
      tracer.stageComplete('post-retrieval', {
        selectedCount: result.selectedCandidates.length,
        droppedCount: result.droppedCandidates.length,
      })
    } else {
      selectedChunks = candidates.map(c => c.chunk)
      tracer.stageComplete('post-retrieval', { selectedCount: selectedChunks.length })
    }

    // Stage 3: Generation
    tracer.stageStart('generation', selectedChunks)
    const answer = await options.generator.generate({ query, chunks: selectedChunks })
    tracer.stageComplete('generation', { answerLength: answer.length })

    const result: RuntimePipelineResult = {
      answer,
      chunks: selectedChunks,
      debugInfo: tracer.getTrace()!,
    }

    tracer.complete({
      retrievalCount: candidates.length,
      selectedCount: selectedChunks.length,
      droppedCount,
      totalTokens: selectedChunks.reduce((sum, c) => sum + (c.tokenCount ?? 0), 0),
    })

    return result
  } catch (err) {
    tracer.error(err instanceof Error ? err : new Error(String(err)))
    throw err
  }
}
```

## 10. 与原独立多包方案的差异

| 维度 | 原 `packages/observability` | 当前 `src/observability/` |
|------|---------------------------|------------------------|
| 包名 | `@rag-sdk/observability` | `@goferbot/rag-sdk` 内部模块 |
| 范围 | 完整的 trace / metric / exporter | 精简版：trace + 默认 console observer |
| 外部集成 | OpenTelemetry / LangSmith / Phoenix | 仅定义 Observer 接口，外部适配后续扩展 |
| 存储 | 支持 trace 后端存储 | 不支持，由外部系统负责 |
| 引用 core | `import from "@rag-sdk/core"` | `import from "../types.js"` |
