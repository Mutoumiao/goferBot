# indexing 模块设计文档

> 离线数据构建模块，负责把原始文档加工成可供 runtime 检索使用的索引数据。

---

## 1. 模块定位

`indexing` 是 `@goferbot/rag-sdk` 内部的离线索引构建模块，职责是组织数据索引流水线：

```text
raw data → document → chunk → vector → store
```

### 1.1 模块目标

1. 统一数据索引流程
2. 降低开发者接入成本
3. 提供默认实现，减少心智负担
4. 保持关键能力可插拔

### 1.2 核心设计原则

1. indexing 是 pipeline orchestration 层，不是第三方 SDK 封装层
2. 关键基础设施由用户决定，流程和默认组件由 SDK 提供
3. 默认实现优先，但必须允许 override
4. 所有跨模块共享模型必须复用 core
5. indexing 不直接依赖具体云服务或向量数据库

---

## 2. 职责边界

### 2.1 indexing 负责什么

1. 定义数据索引流水线的阶段
2. 提供 `runIndexing` 等统一入口
3. 提供一组默认可用组件（chunker / embedder / indexer）
4. 组织 chunker / embedder / store 的调用关系
5. 处理 indexing 过程中的上下文、批次、错误回调、结果汇总

### 2.2 indexing 不负责什么

1. 不负责定义共享领域模型（由 core 负责）
2. 不负责绑定具体向量数据库（由注入的 IVectorStore 负责）
3. 不负责 runtime 检索逻辑
4. 不负责业务侧具体数据源规则

---

## 3. SDK 决定什么，用户决定什么

### 3.1 由 SDK 决定的部分

1. 索引流水线阶段划分
2. 统一执行入口（`runIndexing`）
3. 默认 chunker（`RecursiveCharacterChunker`）
4. 默认 embedder（`OpenAIEmbedder`）
5. 默认错误处理机制与结果汇总格式
6. 默认 batching / pipeline 组织方式

### 3.2 必须由用户决定的部分

1. 向量数据库实现（通过注入 `IVectorStore`）
2. Embedding API 密钥和模型配置
3. 分块参数（chunkSize / chunkOverlap）
4. 文件解析逻辑（输入到 `DocumentSource` 的转换）

### 5.2 OpenAIEmbedder

```typescript
// src/indexing/embedders/openai.ts
import OpenAI from 'openai'
import type { IEmbedder, EmbeddingConfig } from '../../types.js'
import { EmbeddingError, ValidationError } from '../../errors.js'

export class OpenAIEmbedder implements IEmbedder {
  readonly config: Readonly<EmbeddingConfig>
  private readonly client: OpenAI

  constructor(config: EmbeddingConfig) {
    if (config.dimension <= 0) {
      throw new ValidationError('EmbeddingConfig.dimension 必须大于 0')
    }
    this.config = Object.freeze({ ...config })
    this.client = new OpenAI({
      apiKey: config.apiKey,
      baseURL: config.baseUrl,
    })
  }

  async embed(texts: string[]): Promise<number[][]> {
    if (texts.length === 0) {
      throw new ValidationError('texts 不能为空数组')
    }

    const validTexts = texts.filter(t => t.trim().length > 0)
    if (validTexts.length === 0) {
      throw new ValidationError('texts 中所有元素均为空字符串')
    }

    try {
      const response = await this.client.embeddings.create({
        model: this.config.model,
        input: validTexts,
      })

      const vectors = response.data.map(d => d.embedding)

      for (let i = 0; i < vectors.length; i++) {
        if (vectors[i].length !== this.config.dimension) {
          throw new EmbeddingError(
            `第 ${i} 个向量维度不匹配: 期望 ${this.config.dimension}, 实际 ${vectors[i].length}`
          )
        }
      }

      return vectors
    } catch (err) {
      if (err instanceof EmbeddingError || err instanceof ValidationError) {
        throw err
      }
      throw new EmbeddingError('OpenAI Embedding API 调用失败', err)
    }
  }
}
```

**边界行为：**
- 空数组输入抛出 `ValidationError`
- 所有元素为空字符串抛出 `ValidationError`
- 返回向量维度不匹配抛出 `EmbeddingError`
- API 调用失败抛出 `EmbeddingError`（带 cause）

### 5.3 IIndexer 实现（Server 层）

> **架构变更（ADR 0001）**：向量存储使用 PostgreSQL pgvector 扩展。
> SDK 不再提供默认 Indexer 实现，`IIndexer` 由 Server 层独占实现（如 `PrismaVectorIndexer`）。
> 详见 `packages/server/src/processors/indexing/prisma-vector.indexer.ts`。

```typescript
// Server 层实现示例（PrismaVectorIndexer）
import type { IIndexer, Chunk, TokenUsage } from '@goferbot/rag-sdk'
import { PrismaService } from '../database/prisma.service.js'

export class PrismaVectorIndexer implements IIndexer {
  constructor(private readonly prisma: PrismaService) {}

  async index(chunks: Chunk[], vectors: number[][], usage?: TokenUsage[]): Promise<void> {
    if (chunks.length !== vectors.length) {
      throw new ValidationError(`chunks 与 vectors 长度不匹配`)
    }

    // 单事务：同时写入 chunks 元数据 + 向量到 PostgreSQL
    await this.prisma.$transaction(async (tx) => {
      for (let i = 0; i < chunks.length; i++) {
        await tx.$executeRaw`
          INSERT INTO chunks (id, document_id, kb_id, content, token_count, chunk_index, embedding)
          VALUES (${chunks[i].id}::uuid, ${chunks[i].documentId}::uuid, ${chunks[i].kbId}::uuid,
                  ${chunks[i].content}, ${tokenCounts[i]}, ${chunks[i].chunkIndex},
                  ${vectors[i]}::vector)
          ON CONFLICT (id) DO UPDATE SET
            content = EXCLUDED.content,
            token_count = EXCLUDED.token_count,
            embedding = EXCLUDED.embedding
        `
      }
    })
  }
}
```

**边界行为：**
- `chunks` 与 `vectors` 长度不匹配时抛出 `ValidationError`
- 向量维度不匹配时由 PostgreSQL 隐式校验（`::vector` 转换失败）
- 事务失败时抛出 `RAGError`

**VectorRecord 映射（pgvector 方案）：**

| Chunk 字段 | VectorRecord 字段 | PostgreSQL 字段 |
|-----------|------------------|----------------|
| — | `id`（生成 UUID） | `id` |
| `id` | `chunkId` | `chunk_id`（已废弃，pgvector 下 chunk.id 即主键） |
| `kbId` | `kbId` | `kb_id` |
| `documentId` | `fileId` | `document_id` |
| vector[i] | `embedding` | `embedding`（`vector(1536)` 类型） |

## 6. Pipeline 编排

```typescript
// src/indexing/pipeline.ts
import type { DocumentSource, Chunk, IndexingResult, IndexingStage } from '../types.js'
import type { IChunker, IEmbedder, IIndexer } from '../interfaces.js'

export interface IndexingPipelineOptions {
  chunker: IChunker
  embedder: IEmbedder
  indexer: IIndexer
}

export async function runIndexing(
  doc: DocumentSource,
  options: IndexingPipelineOptions
): Promise<IndexingResult> {
  const { chunker, embedder, indexer } = options
  const stages: IndexingStage[] = []

  // Stage 1: Chunking
  stages.push({ name: 'chunking', status: 'running' })
  let chunks: Chunk[]
  try {
    chunks = await chunker.chunk(doc)
    stages[0].status = 'completed'
  } catch (err) {
    stages[0].status = 'failed'
    stages[0].error = err instanceof Error ? err.message : String(err)
    return { chunks: [], vectorCount: 0, stages }
  }

  if (chunks.length === 0) {
    stages[0].status = 'completed'
    return { chunks: [], vectorCount: 0, stages }
  }

  // Stage 2: Embedding
  stages.push({ name: 'embedding', status: 'running' })
  let vectors: number[][]
  try {
    const texts = chunks.map(c => c.content)
    vectors = await embedder.embed(texts)
    stages[1].status = 'completed'
  } catch (err) {
    stages[1].status = 'failed'
    stages[1].error = err instanceof Error ? err.message : String(err)
    return { chunks, vectorCount: 0, stages }
  }

  // Stage 3: Indexing
  stages.push({ name: 'indexing', status: 'running' })
  try {
    await indexer.index(chunks, vectors)
    stages[2].status = 'completed'
  } catch (err) {
    stages[2].status = 'failed'
    stages[2].error = err instanceof Error ? err.message : String(err)
    return { chunks, vectorCount: vectors.length, stages }
  }

  return { chunks, vectorCount: vectors.length, stages }
}
```
