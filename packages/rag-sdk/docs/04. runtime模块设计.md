# runtime 模块设计文档

> 在线检索编排模块，负责查询链路的标准主流程：检索前处理 → 检索 → 检索后处理 → 生成。

---

## 1. 模块定位

`runtime` 是 `@goferbot/rag-sdk` 内部的在线查询编排模块，不是第三方适配层，而是标准在线链路的编排层。

### 1.1 当前现状

截至当前仓库状态：
- core 已完成共享契约（Query / Chunk / RetrievalCandidate / IRetriever / IReranker / IGenerator）
- indexing 已完成离线索引构建主流程
- runtime 当前仍只有接口定义，尚未落地任何真实运行时实现

### 1.2 设计目标

1. 明确 `runtime` 是在线查询编排层
2. 承载完整四阶段链路：pre-retrieval → retrieval → post-retrieval → generation
3. 提供统一上下文与调试信息结构
4. 对 SDK 用户暴露清晰的扩展点

---

## 2. 职责边界

### 2.1 runtime 负责什么

1. 检索前处理（query rewrite、expansion、routing）
2. 检索执行（调用 IRetriever）
3. 检索后处理（rerank、filter、budget trim、diversification、context assembly）
4. 生成调用（调用 IGenerator）
5. 统一上下文传递与调试信息收集

### 2.2 runtime 不负责什么

1. 不定义共享数据模型（由 core 负责）
2. 不实现 indexing 流水线
3. 不绑定具体向量数据库（由注入的 IRetriever 负责）
4. 不实现 LLM 生成（由注入的 IGenerator 负责）

---

## 3. 四阶段运行时模型

```text
pre-retrieval
  -> retrieval
  -> post-retrieval
  -> generation
```

### 3.1 Pre-Retrieval（检索前处理）

可选阶段，当前 MVP 可简化为透传：

```typescript
export interface PreRetrievalOptions {
  rewriteQuery?: boolean      // 是否重写查询
  expandQuery?: boolean       // 是否扩展查询
}
```

### 3.2 Retrieval（检索执行）

调用 `IRetriever.retrieve(query)` 获取候选 RetrievalCandidate。

**混合检索流程**：

```text
Query
  -> 向量检索（Milvus ANN Search） -> 向量候选列表
  -> 关键词检索（PostgreSQL Full-Text Search） -> 关键词候选列表
  -> RRF 融合排序 -> 统一候选列表
```

**RRF（Reciprocal Rank Fusion）公式**：

```text
score_rrf = Σ 1 / (k + rank_i)
```

其中 `k` 为常数（默认 60），`rank_i` 为候选在第 i 个检索结果中的排名。

**设计说明**：
- 向量检索和关键词检索并行执行
- 两套结果通过 RRF 融合为统一分数
- 融合后的结果作为 `RetrievalCandidate[]` 进入 post-retrieval 阶段
- `source` 字段标记候选来源（`'vector'` / `'keyword'` / `'hybrid'`）

### 3.3 Post-Retrieval（检索后处理）

```typescript
export interface RetrievalPostprocessor {
  process(candidates: RetrievalCandidate[], query: Query): Promise<PostRetrievalResult>
}

export interface PostRetrievalResult {
  selectedCandidates: RetrievalCandidate[]
  droppedCandidates: RetrievalCandidate[]
  appliedBudget: number        // 实际使用的 token 预算
  appliedScoreThreshold: number
  selectionTrace: SelectionTrace[]
}

export interface SelectionTrace {
  candidateId: string
  action: 'selected' | 'dropped' | 'reranked'
  reason: string
  stage: 'filtering' | 'reranking' | 'diversification' | 'budget-trim'
}
```

**后处理能力（按优先级分层启用）：**

**第一层：低成本、确定性强的处理**
1. **Result Filtering**：按分数阈值过滤、metadata 过滤
2. **Budget Trim**：按 token 预算截断
3. **Max Chunks 限制**：限制最大返回数量

**第二层：模型打分（可选）**
4. **Reranking**：调用 IReranker 对候选重排序

**第三层：复杂判断（后续扩展）**
5. **Deduplication**：去重
6. **Diversification**：多样性增强（MMR）
7. **Conflict Detection**：冲突检测

### 3.4 Context Assembly（上下文组装）

将选中的 candidates 组织成适合 LLM 阅读的上下文：

export function assembleContext(

**标准格式示例**：

```text
基于以下参考资料回答问题：
---
[1] {chunk.content}
（来源：{documentName}，知识库：{kbName}，章节：{hierarchyPath}）

[2] {chunk.content}
（来源：{documentName}，知识库：{kbName}，章节：{hierarchyPath}）
---

用户问题：{query.original}
```

调用 `IGenerator.generate()` 生成回答。

## 5. 组件设计

### 5.1 HybridRetriever（混合检索入口）

```typescript
// src/runtime/retrievers/hybrid.ts
import type { IRetriever, Query, RetrievalCandidate, HybridSearchOptions } from '../../types.js'
import type { IEmbedder } from '../../interfaces.js'
import type { IVectorStore } from '../../vector-store.js'
import type { IKeywordStore } from './keyword.js'
import { RetrievalError, ValidationError } from '../../errors.js'
import { reciprocalRankFusion } from '../utils/rrf.js'

export interface HybridRetrieverOptions {
  embedder: IEmbedder
  vectorStore: IVectorStore
  keywordStore: IKeywordStore
  /** RRF 融合参数 k，默认 60 */
  rrfK?: number
  /** 向量检索权重，默认 0.5 */
  vectorWeight?: number
  /** 关键词检索权重，默认 0.5 */
  keywordWeight?: number
}

export class HybridRetriever implements IRetriever {
  constructor(private readonly options: HybridRetrieverOptions) {}

  async retrieve(
    query: Query,
    topK: number = 5,
    options?: HybridSearchOptions
  ): Promise<RetrievalCandidate[]> {
    // 1. 校验
    if (!query.original || query.original.trim().length === 0) {
      throw new ValidationError('query.original 不能为空')
    }
    if (!query.kbIds || query.kbIds.length === 0) {
      throw new ValidationError('query.kbIds 不能为空数组')
    }

    const queryText = query.rewritten || query.original
    const rrfK = options?.rrfK ?? this.options.rrfK ?? 60

    try {
      // 2. 并行执行向量检索和关键词检索
      const [vectorCandidates, keywordCandidates] = await Promise.all([
        this.vectorSearch(queryText, query.kbIds, topK * 2),
        this.keywordSearch(queryText, query.kbIds, topK * 2),
      ])

      // 3. RRF 融合
      const fused = reciprocalRankFusion(
        [vectorCandidates, keywordCandidates],
        rrfK
      )

      return fused.slice(0, topK)
    } catch (err) {
      if (err instanceof ValidationError) throw err
      throw new RetrievalError('混合检索执行失败', err)
    }
  }

  private async vectorSearch(queryText: string, kbIds: string[], topK: number): Promise<RetrievalCandidate[]> {
    const queryVectors = await this.options.embedder.embed([queryText])
    const queryVector = queryVectors[0]

    const candidates: RetrievalCandidate[] = []
    for (const kbId of kbIds) {
      const results = await this.options.vectorStore.searchVectors(queryVector, {
        topK: Math.ceil(topK / kbIds.length),
        filter: { kbId },
      })

      for (const result of results) {
        candidates.push({
          chunk: {
            id: result.chunkId,
            documentId: '', // 由 server 补充
            kbId,
            content: '',    // 由 server 补充
            chunkIndex: 0,
          } as Chunk,
          score: result.score,
          source: 'vector',
          route: `vector:${kbId}`,
        })
      }
    }
    return candidates
  }

  private async keywordSearch(queryText: string, kbIds: string[], topK: number): Promise<RetrievalCandidate[]> {
    const results = await this.options.keywordStore.searchKeywords(queryText, {
      kbIds,
      topK,
    })

    return results.map(r => ({
      chunk: {
        id: r.chunkId,
        documentId: '', // 由 server 补充
        kbId: r.kbId,
        content: '',    // 由 server 补充
        chunkIndex: 0,
      } as Chunk,
      score: r.score,
      source: 'keyword',
      route: `keyword:${r.kbId}`,
    }))
  }
}
```

**边界行为：**
- `query.original` 为空字符串时抛出 `ValidationError`
- `query.kbIds` 为空数组时抛出 `ValidationError`
- 向量库或关键词库查询失败时抛出 `RetrievalError`
- 并行执行，任一失败不影响另一结果（失败方返回空列表，降级为单路检索）

### 5.3 RRF 融合算法

```typescript
// src/runtime/utils/rrf.ts
import type { RetrievalCandidate } from '../../types.js'

/**
 * Reciprocal Rank Fusion（倒数排名融合）。
 * 
 * 将多路检索结果融合为统一排序列表。
 * 
 * @param lists - 多路候选列表，每路已按分数降序排列
 * @param k - RRF 常数，默认 60（防止低排名项分数过高）
 * @returns 按融合分数降序排列的候选列表
 */
export function reciprocalRankFusion(
  lists: RetrievalCandidate[][],
  k: number = 60
): RetrievalCandidate[] {
  const scores = new Map<string, number>()
  const candidates = new Map<string, RetrievalCandidate>()

  for (const list of lists) {
    for (let rank = 0; rank < list.length; rank++) {
      const candidate = list[rank]
      const id = candidate.chunk.id
      const rrfScore = 1 / (k + rank + 1)

      scores.set(id, (scores.get(id) ?? 0) + rrfScore)
      // 保留最高分的候选版本
      if (!candidates.has(id) || candidate.score > (candidates.get(id)!.score)) {
        candidates.set(id, { ...candidate, source: 'hybrid' as const })
      }
    }
  }

  return Array.from(candidates.values())
    .map(c => ({
      ...c,
      score: scores.get(c.chunk.id) ?? 0,
    }))
    .sort((a, b) => b.score - a.score)
}
```

**算法说明**：
- 每路检索结果独立排名，计算 `1 / (k + rank)`
- 同一候选在多路中出现则分数累加
- `k=60` 是业界常用值，平衡高排名和低排名的差异
- 融合后 `source` 标记为 `'hybrid'`

### 5.2 DefaultRetrievalPostprocessor

```typescript
// src/runtime/postprocessors/default.ts
import type { RetrievalPostprocessor, RetrievalCandidate, PostRetrievalResult, SelectionTrace } from './types.js'
import type { Query } from '../../types.js'
import type { IReranker } from '../../interfaces.js'

export interface DefaultPostprocessorOptions {
  scoreThreshold?: number      // 默认 0.5
  maxTokens?: number           // 默认 4000
  maxChunks?: number           // 默认 10
  reranker?: IReranker         // 可选 reranker
  diversityLambda?: number     // MMR 参数（后续扩展）
  deduplicate?: boolean        // 是否去重（后续扩展）
}

export class DefaultRetrievalPostprocessor implements RetrievalPostprocessor {
  constructor(private readonly options: DefaultPostprocessorOptions = {}) {}

  async process(candidates: RetrievalCandidate[], query: Query): Promise<PostRetrievalResult> {
    const traces: SelectionTrace[] = []
    let working = [...candidates]

    // Stage 1: Filtering（分数阈值）
    const threshold = this.options.scoreThreshold ?? 0.5
    const filtered = working.filter(c => {
      const keep = c.score >= threshold
      if (!keep) {
        traces.push({
          candidateId: c.chunk.id,
          action: 'dropped',
          reason: `分数 ${c.score.toFixed(4)} 低于阈值 ${threshold}`,
          stage: 'filtering',
        })
      }
      return keep
    })
    working = filtered

    // Stage 2: Reranking（可选）
    if (this.options.reranker && working.length > 0) {
      working = await this.options.reranker.rerank(working, query)
      for (const c of working) {
        traces.push({
          candidateId: c.chunk.id,
          action: 'reranked',
          reason: '经过 reranker 重排序',
          stage: 'reranking',
        })
      }
    }

    // Stage 3: Budget Trim（token 预算 + 最大 chunks）
    const maxChunks = this.options.maxChunks ?? 10
    const maxTokens = this.options.maxTokens ?? 4000
    let tokenCount = 0
    const selected: RetrievalCandidate[] = []
    const dropped: RetrievalCandidate[] = []

    for (const c of working) {
      const chunkTokens = c.chunk.tokenCount ?? c.chunk.content.length // 粗略估计
      if (selected.length >= maxChunks || tokenCount + chunkTokens > maxTokens) {
        dropped.push(c)
        traces.push({
          candidateId: c.chunk.id,
          action: 'dropped',
          reason: selected.length >= maxChunks
            ? `超出最大 chunks 限制 ${maxChunks}`
            : `超出 token 预算 ${maxTokens}`,
          stage: 'budget-trim',
        })
      } else {
        selected.push(c)
        tokenCount += chunkTokens
      }
    }

    return {
      selectedCandidates: selected,
      droppedCandidates: [...dropped, ...candidates.filter(c => !working.includes(c) && !dropped.includes(c))],
      appliedBudget: tokenCount,
      appliedScoreThreshold: threshold,
      selectionTrace: traces,
    }
  }
}
```

## 6. Pipeline 编排

```typescript
// src/runtime/pipeline.ts
import type { Query, Chunk, RuntimePipelineResult, RuntimeDebugInfo, RuntimeStage } from '../types.js'
import type { IRetriever, IGenerator, IReranker } from '../interfaces.js'
import type { RetrievalPostprocessor } from './postprocessors/types.js'

export interface RuntimePipelineOptions {
  retriever: IRetriever
  postprocessor?: RetrievalPostprocessor
  generator: IGenerator
}

export async function runRetrievalPipeline(
  query: Query,
  options: RuntimePipelineOptions
): Promise<RuntimePipelineResult> {
  const startTime = Date.now()
  const stages: RuntimeStage[] = []
  const { retriever, postprocessor, generator } = options

  // Stage 1: Retrieval
  stages.push({ name: 'retrieval', startTime: Date.now(), endTime: 0, input: query, output: null })
  let candidates = await retriever.retrieve(query)
  stages[0].endTime = Date.now()
  stages[0].output = { count: candidates.length }

  // Stage 2: Post-Retrieval
  stages.push({ name: 'post-retrieval', startTime: Date.now(), endTime: 0, input: candidates, output: null })
  let selectedChunks: Chunk[]
  let droppedCount = 0
  if (postprocessor) {
    const result = await postprocessor.process(candidates, query)
    selectedChunks = result.selectedCandidates.map(c => c.chunk)
    droppedCount = result.droppedCandidates.length
    stages[1].output = {
      selectedCount: result.selectedCandidates.length,
      droppedCount: result.droppedCandidates.length,
      appliedBudget: result.appliedBudget,
    }
  } else {
    selectedChunks = candidates.map(c => c.chunk)
  }
  stages[1].endTime = Date.now()

  // Stage 3: Generation
  stages.push({ name: 'generation', startTime: Date.now(), endTime: 0, input: selectedChunks, output: null })
  const answer = await generator.generate({ query, chunks: selectedChunks })
  stages[2].endTime = Date.now()
  stages[2].output = { answerLength: answer.length }

  const debugInfo: RuntimeDebugInfo = {
    traceId: crypto.randomUUID(),
    query,
    stages,
    metrics: {
      retrievalCount: candidates.length,
      selectedCount: selectedChunks.length,
      droppedCount,
      totalTokens: selectedChunks.reduce((sum, c) => sum + (c.tokenCount ?? 0), 0),
      latencyMs: Date.now() - startTime,
    },
  }

  return {
    answer,
    chunks: selectedChunks,
    debugInfo,
  }
}
```

## 10. 关键变更记录

| 变更项 | 原设计 | 新设计 | 原因 |
|--------|--------|--------|------|
| IRetriever 入参 | `query: string, kbIds: string[]` | `query: Query` | 统一查询抽象，支持 rewrite/expansion |
| IRetriever 返回值 | `ChunkWithScore[]` | `RetrievalCandidate[]` | 支持 source、route、metadata |
| 检索策略 | 仅向量检索（`MilvusRetriever`） | 混合检索（`HybridRetriever`：向量 + 关键词 + RRF） | 主流检索方案，提升召回率 |
| 关键词检索 | 无 | 新增 `IKeywordStore` 接口 | 支持 PostgreSQL Full-Text Search |
| RRF 融合 | 无 | 新增 `reciprocalRankFusion` | 融合多路检索结果 |
| Postprocessor | 仅分数过滤 + 预算截断 | 分层处理：filter → rerank → budget trim | 支持更丰富的检索后处理能力 |
| SelectionTrace | 无 | 新增，记录每步操作原因 | 支持可观测性和调试 |
| Context Assembly | 无独立函数 | 新增 `assembleContext` | 支持多种格式和来源标注 |
| 引用路径 | `../../core/index.js` | `../../types.js` | core 扁平化到 src 根目录 |
